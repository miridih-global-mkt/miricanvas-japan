# 가이드라인 — 껍데기(데모) 버전

작성일: 2026-05-20
범위: `index.html` 단일 정적 파일로 구현한 **UI 프로토타입**.
목적: 실제 동작 없이 "이런 느낌의 사이트가 될 거야"를 팀/이해관계자에게 보여주기 위함.

---

## 0. 한 줄 요약

「ミーキャンPPTで私の○○自慢」 콘테스트 사이트의 **시각적 골격**을 보여주는 정적 페이지.
데이터는 모두 하드코딩, 클릭은 시각적 피드백만, 백엔드 없음.

## 1. 구현된 것 (보여지는 것)

| 영역 | 내용 |
|---|---|
| 헤더 | 로고 / 현재 시리즈명 + 드롭다운 / `応募する` 버튼. 스크롤 시 sticky |
| 히어로 | 시리즈명(한자 명조체) + 설명 + 응모/기간/투표 통계 + CTA 2종(`応募する` / `自分の作品をさがす`) |
| 랭킹 섹션 | 탭: **累計ランキング / 今月のランキング** · 상위 3위 포디움 카드 + 4-10위 리스트 |
| 갤러리 | 정렬 토글(人気順 / 新着順) · 카드 그리드 (썸네일 + 제목 + 작가 + 좋아요수 + 투표 버튼 + 게시일) |
| 다른 시리즈 | 4개 카드(終了 / 開催中 / 近日) — 사이드바 대신 페이지 하단에 배치 |
| 푸터 | 카피라이트 / 규약 / 문의 링크(빈 링크) |
| 모달 3종 | ① 응모 폼  ② 내 작품 찾기  ③ 시리즈 picker |

## 2. 의도적으로 빼놓은 것 (껍데기이므로 X)

- 실제 투표 카운트 증가 / 서버 저장
- 중복 투표 방지 (LocalStorage / IP / OAuth)
- 이메일 매직링크 발송
- 응모 폼 제출 후 검증
- OG 이미지 자동 추출
- 어드민 페이지
- 시리즈 간 실제 라우팅 (전부 같은 페이지)

→ 이건 전부 `GUIDELINE_full.md` 쪽 책임.

## 3. UI 톤 & 디자인 원칙

**컨셉**: 세련된 일본 잡지 미감. 여백(間), 얇은 헤어라인 보더, 단일 액센트 컬러.

### 컬러
| Token | Hex | 용도 |
|---|---|---|
| `--bg` | `#FAFAF7` | 페이지 배경 (살짝 따뜻한 오프화이트) |
| `--card` | `#FFFFFF` | 카드 / 모달 |
| `--border` | `#E8E6DF` | 헤어라인 보더 |
| `--text` | `#1A1A1A` | 본문 |
| `--text-sub` | `#6B6B6B` | 보조 |
| `--text-mute` | `#9B9B9B` | 메타 (날짜, 라벨) |
| `--accent` | `#BC002D` | 일본의 적(赤). 하이라이트, 좋아요, 액션 |
| `--gold / silver / bronze` | `#C9A86A / #B8B8B8 / #C08552` | 1·2·3등 메달 |

### 타이포
- **제목 / 숫자**: `Zen Old Mincho` (없으면 `Noto Serif JP`) — 한자 명조체. 격조 있는 느낌.
- **본문 / UI**: `Noto Sans JP` — 깔끔, 가독성.
- 자간 약간 넓게 (`letter-spacing: 0.04~0.1em`) — 일본 UI 특유의 정돈된 느낌.

### 형태
- 모서리 거의 직각 (`border-radius: 2px`만). 일본 디자인은 둥근 모서리를 잘 안 씀.
- 그림자 거의 X. 헤어라인(`1px`)으로 분리.
- 액센트 라인은 좌측 2px 세로 막대 또는 하단 1px 밑줄.

### 반응형 breakpoint
| 화면폭 | 갤러리 | 포디움 |
|---|---|---|
| ≥ 1024px | 3 columns | 3 columns |
| 600~1023px | 2 columns | 1 column |
| < 600px | 1 column | 1 column |

모바일에서 헤더는 로고만 남기고 `応募する` CTA는 ●(원형) 아이콘 버튼으로 축소.

## 4. 인터랙션 (껍데기 수준)

| 요소 | 동작 |
|---|---|
| 헤더 시리즈 드롭다운 | 클릭 시 모달 오픈, 4개 시리즈 카드 표시 |
| `応募する` 버튼 | 응모 폼 모달 (필드 보이지만 제출 누르면 "シェル版では未送信です" 토스트) |
| `自分の作品をさがす` | 모달 + 이메일 입력 → "확인" 누르면 데모용 결과 1개 표시 |
| 좋아요 (♥) | 클릭하면 채워진 ♥로 변경 + 카운트 +1 (페이지 새로고침 시 리셋) |
| 랭킹 탭 | 累計/今月 전환 시 상위 3위 데이터만 바꿔치기 |
| 정렬 탭 | 人気順/新着順 전환 시 카드 순서 재정렬 |

## 5. 파일 구조

```
[이벤트]/
  index.html              ← 이 한 파일이 데모 전체. 더블클릭으로 열림
  GUIDELINE_shell.md      ← 본 문서
  GUIDELINE_full.md       ← 실제 구축용 풀스펙
```

CSS·JS·데이터 모두 `index.html` 안에 인라인. 외부 의존성은 Google Fonts와 picsum.photos(썸네일 더미)뿐.

## 7. 데모 보는 법

`index.html` 파일을 더블클릭하거나 브라우저에 드래그. 인터넷 연결만 있으면 폰트와 더미 이미지가 로드됨.

## 8. 데모 → 실구현 이행 시 체크포인트

데모를 그대로 가져다 본 구현의 마크업 베이스로 써도 좋지만, 다음은 반드시 검토:

- 모달은 접근성(focus trap, ESC 닫기)을 다시 작업해야 함
- 좋아요 버튼은 OAuth 연동 + 낙관적 UI로 재구현
- 썸네일은 picsum 더미 → 미캔 OG 추출로 교체
- 한자 명조체 폰트는 라이선스 확인 후 self-host (CLS 줄이기)
