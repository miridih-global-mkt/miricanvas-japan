# 메트릭 선정 의사결정 근거 문서

## Executive Summary

CRM Global Canvas Dashboard의 핵심 KPI 메트릭을 **9개로 선정**했다. 본 문서는 선정 절차·기준·근거를 설명한다.

### 선정된 핵심 9개 메트릭

| # | 한국어명 | 영어 ID | 핵심 역할 |
|---|---|---|---|
| 1 | 유저수 | user_count | 모수 (cell 크기 표시) |
| 2 | 로그인평균일수D3 | LPU72h | L2 segment 분석 강세 (L2 #3) |
| 3 | 로그인평균일수D7 | LPU168h | **Universal champion** (L1 #7, L2 #4) |
| 4 | (총)로그인평균일수 | LPU_total | L1 강함, 장기 view (L1 #5) |
| 5 | 로그인2일이상비율D3 | L72h_ge2 | **Universal strong** (L1 #4, L2 #5) |
| 6 | 로그인3일이상비율D7 | L168h_ge3 | **L1 최강** (L1 #3) |
| 7 | 디자인평균개수D3 | DPU72h | L2 디자인 차원 (L2 #12) |
| 8 | 디자인평균개수D7 | DPU168h | 디자인 차원 D7 비교 |
| 9 | 포스트W1재로그인률 | LRateReturn | CRM 장기 anchor (모니터링용) |

추가: **컨트롤 그룹이 없는 페이지에선 열람률(open_rate)도 함께 표시** (메일 진단용).

---

## 1. 메트릭 선정 가이드라인 (사전 정의)

본 선정은 별도 가이드라인(부록 A: 메트릭 선정 가이드라인) 기준에 따른다. 핵심 원칙:

### 좋은 메트릭의 정의
**메일 효과가 있을 때 명확히 반응하고, 없을 때 흔들리지 않는 메트릭**.

통계적 표현: **신호 대 잡음 비율(Signal-to-Noise Ratio) z값**이 큰 메트릭.

### 평가 기준 (1순위: 일관성, 2순위: 명료성)
- **일관성**: 일별 신호 부호 일관성 70% 이상이면 우연이 아닌 것으로 간주
- **명료성**: |z| ≥ 2 이상이면 잡음 대비 신호가 충분히 큼

### 분석 층위
- **L1 (캔버스 단위)**: 캔버스 전체에서 통제군 vs 메일수신군 비교
- **L2 (캔버스 × user_type)**: 6개 user_type 세그먼트별 비교
- L3 (캔버스 × ad_campaign)는 1차 분석에서 보류

---

## 2. 선정 절차

### Step 1: 후보 메트릭 도출 (44개)
- 원본 데이터 컬럼(35개) + 대시보드 measure + 분석상 파생 + CRM 관점
- 시간 윈도우(D1·D3·D7·총) × 측정 방식(binary·평균·threshold·ratio) 모두 검토
- 6개 카테고리, 44개 후보 채택
- 약하거나 중복되는 12개는 후보에서 제외 (LRateW1, LPU×100, DPU log 등)

### Step 2: L1·L2 ranking 산출 (50개)
- 44개 + 보완 검증용 LPU/DPU raw 4+4 추가 = 50개 분석
- 주별 코호트 기반 신호/잡음 비율 측정
- 모수 임계: Control ≥ 100 (loose threshold)

### Step 3: 핵심 메트릭 선정 (9개)

선정 기준 (우선순위):
1. **Universal robust** (L1·L2 양 층위에서 강함)
2. **L2 메인** (segment 분석 강세 — 본 데이터에서 사용자별 차별 효과 발견됨)
3. **L1 only 강세** (큰 그림에서 강한 신호 보존)
4. **시간 윈도우 다양성** (단기·중기·장기 cover)
5. **CRM 장기 anchor** (데이터 누적 시 가치 강화 예상)

## 3. 선정 9개 메트릭 — 항목별 근거

### ⭐ #1 유저수 (user_count)
- **역할**: 모수 표시
- **선정 근거**: 모든 cell의 표본 크기 확인 필수. 통계 신뢰성 판단 기초

### ⭐ #2 로그인평균일수D3 (LPU72h) — L2 강세
- **L1 #14, L2 #3**
- **선정 근거**: L2 segment 분석에서 두 번째로 강한 신호. 사용자별 차별 효과를 가장 잘 잡음
- **시간 윈도우**: 72h(=3일) — 메일 직후 단기 반응 측정

### ⭐⭐⭐ #3 로그인평균일수D7 (LPU168h) — **Universal Champion**
- **L1 #7, L2 #4** — 양 층위 모두 top 7
- **선정 근거**: **모든 분석 차원에서 가장 안정적인 메트릭**. 어떤 입자도에서 봐도 메일 효과를 robust하게 측정
- **시간 윈도우**: 168h(=7일) — CRM 업계 표준 활성화 윈도우
- **핵심 KPI 후보 1순위**

### ⭐ #4 (총)로그인평균일수 (LPU_total) — L1 강
- **L1 #5, L2 #10**
- **선정 근거**: L1 5위로 매우 강. **누적 view**를 제공하면서도 통계적으로 안정
- **방어 논리**: "단기 metric만 본다"는 비판에 대한 답 — 누적 시점에서도 강함을 입증

### ⭐⭐ #5 로그인2일이상비율D3 (L72h_ge2) — Universal Strong
- **L1 #4, L2 #5** — 두 층위 모두 top 5
- **선정 근거**: 3일 안에 2일 이상 들어온 유저 비율 = **재방문 의지** 측정. 일회성 방문자 배제하고 진짜 활성화된 유저 식별
- **Universal 강세**: 어떤 입자도에서도 강한 신호

### ⭐⭐⭐ #6 로그인3일이상비율D7 (L168h_ge3) — **L1 최강**
- **L1 #3** (전체 ranking 1위 자리), L2 #9
- **선정 근거**: L1에서 **A category 제외 전체 1위**. 7일 중 3일 이상 로그인 = **본격 활성화** 측정
- **L1 메인 의사결정용 핵심**

### ⭐ #7 디자인평균개수D3 (DPU72h) — L2 디자인 차원
- **L1 #26, L2 #12**
- **선정 근거**: 디자인 관련 메트릭 중 **L2에서 가장 강함**. 디자인 차원을 위젯에서 살리기 위한 단 하나의 디자인 metric
- **주의**: L1에선 약함(#26). 디자인 행동은 segment-specific하다는 발견의 결과

### ⭐ #8 디자인평균개수D7 (DPU168h) — 디자인 D7 비교
- **L1 #43, L2 #17**
- **선정 근거**: D3와 짝지어 디자인 행동의 시간상 변화 추적. L2에선 mid 수준의 신호 있음
- **주의**: L1에서 매우 약함. 사용자 사이에서 디자인 효과가 평균에 묻히는 패턴 확인

### ⭐ #9 포스트W1재로그인률 (LRateReturn) — CRM Anchor
- **L1 #39, L2 #47**
- **선정 근거**: 통계적으로는 약하지만 **CRM 관점에서 핵심**. 가입 후 1주 이후 재방문한 유저 비율 — "서비스 안착" 측정
- **현 한계**: 관측 기간 4~6주로 후기 코호트가 자연히 낮게 측정됨
- **향후 가치**: 데이터 누적(3개월+) 시 강한 leading indicator로 부상 예상
- **운영 방침**: 모니터링용으로 운영, 주요 의사결정은 다른 metric으로

## 4. 핵심 메트릭 그룹 분석

선정된 9개를 역할별로 묶으면:

### 🥇 1순위 — Universal Champion
**로그인평균일수D7 (LPU168h)** — 단일 메트릭으로 메일 효과 모니터링 가능. 모든 종류 보고서·논의에서 1번 metric.

### 🥈 2순위 — L1·L2 Pair
- **로그인3일이상비율D7** (L1 #3) + **로그인2일이상비율D3** (L1 #4, L2 #5)
- 두 metric으로 L1·L2 모두 강신호 보장

### 🥉 3순위 — 시간 차원 보완
- **로그인평균일수D3** (L2 강세) + **(총)로그인평균일수** (L1 강, 누적)
- LPU 시리즈 4개로 단기·중기·장기 모두 cover

### 보조 — 디자인 차원 + CRM Anchor
- **디자인평균개수 D3·D7**: 디자인 차원 보존
- **포스트W1재로그인률**: 장기 retention anchor

## 5. 의도적으로 제외한 메트릭과 사유

### 제외 ① 단순 binary 로그인비율 (LRate 시리즈)
- **이유**: 정보량 적음 (0/1만), 모든 시간 윈도우에서 mid-pack
  - LRate24h L1#15, L2#7
  - LRate72h L1#17, L2#11
  - LRate168h L1#12, L2#17
- **사실상 LPU의 ge1 threshold 케이스** — 더 정보 많은 threshold·평균 변형이 있는데 굳이 binary 채택 의미 없음
- **운영 보완**: 의사소통상 "% 도달" 필요하면 별도 슬라이드에서 보조 표시

### 제외 ② 디자인 관련 대부분 (디자인 깊이, 디자인 binary, 조합, Stickiness)
- **이유**: 디자인 행동 전반이 메일 효과 측정에 약함
  - 디자인복수생성비율: L2 #25 ~ #48
  - 디자인비율 binary: L2 #19 ~ #38
  - Stickiness 시리즈: L2 #32 ~ #50 (꼴찌)
  - 조합 LD·LDeep: L2 #34 ~ #36
- **메일이 디자인 행동을 직접 유도하지 못한다**는 발견이 핵심. 측정 도구 문제가 아니라 실제 효과의 한계
- **유지된 디자인**: DPU72h(L2 #12), DPU168h(L2 #17)만 살림 — 디자인 차원 최소 1~2개는 보존

### 제외 ③ 누적(총) binary·threshold 대부분
- **이유**: 코호트 편향. 가입 일찍 한 유저가 누적기간 길어 자연히 누적값 높아짐 — 메일 효과와 무관한 잡음
  - (총)디자인비율 L1 #48
  - (총)디자인평균개수 L1 #46
  - (총)로그인일당디자인수 L1 #49
- **예외 유지**: (총)로그인평균일수만 채택 (L1 #5로 매우 강 — 코호트 편향에도 신호 살아남음)

### 제외 ④ Activ_AUC, LDeep168h (L1만 강, L2 약)
- **이유**: L1 #9, #10으로 강하지만 L2에선 #35, #43으로 약함
- **L2 메인 전략과 어긋남** — segment 분석 못함
- 만약 L1만 보면 강력한 후보들이지만, L2 메인 원칙상 제외

## 7. 메트릭 운영 방침

### 메인 의사결정
1. **단일 KPI 보고**: 로그인평균일수D7 우선
2. **L1 분석**: 로그인3일이상비율D7 + (총)로그인평균일수 + 로그인평균일수D7
3. **L2 분석**: 로그인평균일수D3·D7 + 로그인2일이상비율D3 + 디자인평균개수D3

### 운영 주기
- **주별**: 모든 9개 metric 추적
- **분기별**: 포스트W1재로그인률 재평가 (데이터 누적 시 신호 강화 예상)

### 의사결정 임계
- **강한 양효과**: |z| > 3 + 일관성 100% → 즉시 채택
- **강한 역효과**: |z| > 2 + 일관성 100% → 즉시 배제 또는 정성 분석
- **약신호**: |z| 1~2, 일관성 50~70% → 모니터링 + 데이터 누적 대기

### Stakeholder 응대
- "% 로그인" 질문 시 → 로그인평균일수가 정보량 많고 통계 강함을 설명
- "최종 결과만" 요구 시 → (총)로그인평균일수 + 포스트W1재로그인률로 보완 (D7은 leading indicator)

## 8. 추가 분석 권장 (향후)

1. **역효과 캠페인 분석** — D1 aip_curiosity, D2 image_bgremove 등 강한 역효과 신호 정밀 분석 (별도 진행 예정)
2. **L3(광고 그루핑) 분석** — 퍼포먼스 마케터에게 ad_campaign 그루핑 요청 후 진행
3. **장기 retention 재분석** — 데이터 3개월+ 누적 후 포스트W1재로그인률 ranking 재평가
4. **세그먼트별 메일 차별화 A/B** — L2 분석에서 segment-specific 효과 강하게 확인됨에 따라
