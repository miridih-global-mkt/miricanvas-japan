사용자께서 "잘 만들어졌는지" 어떻게 확인할 수 있는지의 기준선.

① 에러 처리 정책

상황 동작
저장 실패 (디스크 가득, 권한 등) 상단에 빨간 알림 배너 + 저장 버튼은 활성 유지 (재시도 가능). 데이터는 메모리에 보존
로딩 실패 (JSON 손상) "파일이 손상되었습니다. 백업에서 복구하시려면 Claude Code에 명령하세요" 안내
이미지 URL 깨짐 렌더링 화면에서 placeholder 이미지로 fallback (편집 화면엔 빨간 ⚠ 아이콘 표시)
CTA href 비어있음 편집 화면에 노란 ⚠ 표시 (저장은 가능, 발송 전 채울 사항)
서버 다운 (브라우저는 떠있는데 Flask 죽음) 상단 빨간 배너 "서버 연결 끊김. 콘솔에서 python app.py 다시 실행해주세요"
메일 ID 충돌 (동시 생성 등) UUID 사용 → 충돌 가능성 사실상 없음. 발생 시 재생성

② 테스트 전략

🎯 핵심: 렌더러 정합성 테스트

module_catalog_annotated.html 원본을 빌더로 그대로 재현 → 렌더 결과 vs 원본 HTML 자동 비교.

비교 기준:
• 공백/주석 정규화 후 텍스트 diff
• 인라인 스타일 키 순서 무시
• id 속성 무시

위치: _builder/tests/renderer_fidelity_test.py
실행: python -m pytest

📋 보조: 매뉴얼 테스트 체크리스트

릴리스 전 사용자(또는 Claude)가 직접 클릭하며 확인:
• 새 메일 생성 → 카탈로그 모듈 다 들어있나
• 섹션/모듈 추가·삭제·드래그 동작
• 비주얼/코드 토글
• 저장 → 파일 생성 확인
• 백업 폴더에 이전 버전 있나
• Ctrl+Z 동작
• 비교 탭 폭 토글 (데스크톱/모바일)

체크리스트 파일: _builder/tests/manual.md

③ 완료 기준 (Acceptance Criteria)

아래 모든 항목이 ✅이면 v1 완료:

  • 핵심 흐름: 새 메일 만들기 → 모듈 편집 → 저장 → 재오픈 → 동일하게 보임
  • 렌더러 정합성: 카탈로그 원본을 그대로 재현 시 HTML diff 0 (위 자동 테스트 통과)
  • 이메일 세이프 HTML: 출력에 <div>, style=... 같은 비허용 태그/속성 없음 (sanitizer 검증)
  • 3 탭 모두 동작: 소개·제작·비교 탭 모두 의도대로 (스펙 5~6번)
  • 편집 모드 락: 편집 중 다른 메일 편집 버튼 누르면 차단 팝업
  • 비교 탭: 가로 스크롤 + 폭 토글 (데스크톱 600 / 모바일 375 / 맞춤)
  • 백업: 저장 시 직전 3개가 mails/.backup/에 보관됨 (수동 확인)
  • Undo: Ctrl+Z 30단계 동작
  • 실 이메일 클라이언트 1회 송출 테스트: 코드 복사 → 본인 Gmail로 보내기 → 의도대로 렌더되는지 육안 확인
