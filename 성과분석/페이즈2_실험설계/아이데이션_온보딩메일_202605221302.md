## Track B — 세그먼트별 인사이트 → 새 메일 방향

### STUDENT: 기능 설명만 통한다 / 감정 어필 전면 금지

| 뭐가 안 통함 | 뭐가 통함 |

| D1 전체 (공감/AI호기심/리뷰) — net -29~-52 | D2 data_graph (+39), D2 font (+24), D2 data_table (+23) |
| "디자인을 더 잘 할 수 있어요" | "차트 이렇게 만들어요" |
| 소셜 프루프 | 구체적 기능 사용법 |

**왜**: 학생은 실용적 도구로 인식. 정서·소셜 어필 = 부담. 빠르게 끝나는 기능 팁만 환영.

**새 메일 방향**:
- 완전히 tutorials/how-to 포맷
- 제목: "〜する方法" 패턴 고정
- 감정 단어 제거
- data_graph가 가장 강했으니 데이터 시각화 기능 정면 다루기

**새 메일 후보**:
- "データをグラフにする3ステップ" — D2 data_graph 강점 활용 튜토리얼
- "フォントを変えるだけで資料の印象が変わる" — D2 font 강점 활용
- "レポートを10分で完成させる方法" — 작업 완료 프레임 (감정 없음)

## Track C — 소구점 백서 기반 새 HTML 아이디어

### 아이디어 1 — "生成 → 編集" 황금 루트 체험 유도 메일
**타겟**: NULL / UNKNOWN (가입 후 아직 아무것도 안 한 유저)  
**타이밍**: D+1  
**소구점 기반**: 백서 핵심 인사이트 — "템플릿 생성 + 편집 두 경험을 한 유저 = 황금 유저"

```
구조:
[헤더] 「MiriCanvasを使いこなす人はみな、このルートを通っています」
[2단계 플로우 카드]
  STEP 1: テンプレートを選んで生成（3分）
           → 주제 입력 → AI가 30장 완성
  STEP 2: 1つだけ自分用に変える
           → 로고 색상 or 텍스트 1개 수정
[CTA] 「今すぐ1枚を自分のものに」
[하단] 「ここまでできた人の継続率は、そうでない人の3倍です」
```

**HTML 포인트**: 2단계 플로우를 좌→우 화살표. STEP1 배경 mc-soft(민트), STEP2 배경 네이비. 여정감 강조.

### 아이디어 4 — "AI가 이렇게 만들었다" 결과물 임팩트 메일
**타겟**: NULL / UNKNOWN (AI 기능 미경험 유저)  
**타이밍**: D+2  
**소구점 기반**: 백서 1.1 — "텍스트 1행 → 30장. 3분 이내."

```
구조:
[헤더] 「テキスト1行、3分後にどうなったか」
[Before/After]
  BEFORE 입력: 「新しいコーヒーショップのオープン案内を作って」(회색 박스)

### 아이디어 5 — Smart Block 마법 체험 메일
**타겟**: COMPANY / UNKNOWN  
**타이밍**: JAS 시리즈 중간 삽입 (D+8~11)  
**소구점 기반**: 경쟁사 없는 Smart Block (백서 강점 ⑥)

```
구조:
[헤더] 「内容を変えたら、レイアウトが崩れた経験ありませんか？」
[Before/After 2단 카드]
  Without Smart Block: 텍스트 늘렸더니 이미지 밀리고 정렬 무너진 슬라이드
  With Smart Block: 텍스트 늘려도 자동 재배치, 깔끔하게 유지
[설명 박스]
  「MiriCanvasのSmart Blockは、内容を変えるたびに
   レイアウトを自動で再計算します。崩れません。」
[솔직한 주의사항] ※ Smart Block解除はいつでも可能。一部操作はCtrl+Zが効かない場合があります。
[CTA] 「Smart Blockを試してみる」
```

**HTML 포인트**: Before=와장창 슬라이드, After=정갈한 슬라이드 나란히. 주의사항 솔직 표기 → 신뢰도 상승 (백서 "전략적 × 원칙" 온보딩 적용).

### 아이디어 6 — "첫 작품 완성" 활성화 메일 (가장 단순)

**타이밍**: D+3 (가입 후 미활성 조건 분기)  
**소구점 기반**: 첫 생성 경험 유저의 리텐션 압도적으로 높음

```
구조:
[헤더] 「最初の1枚を作ると、何かが変わります」
[진행도 바 CSS only]
  [가입] → [첫 생성 ★] → [편집] → [완성]
            ↑ あなたは今ここ
[카피]
  「データによると、最初の1枚を作ったユーザーの
   多くが翌日も戻ってきています。理由は簡単です。
   自分で作ったものは、見返したくなるから。」
