# CRM 진성고객 KPI 기반 일본 메일 재분석 v2 — 로그인 + 다운로드 통합

**작성**: 2026-05-20 (v1 갱신)
**v1과 차이**: 디자인 다운로드 metric 추가. D0 download 가설 검증. v1의 "D2가 retention driver" 결론을 download 관점에서 재평가 → 약화.

---

## 0. v1에서 v2로 — 무엇이 바뀌었나

| 항목 | v1 (로그인 only) | v2 (로그인+다운로드) |
|---|---|---|
| M1 최강 proxy | BW1 login (+42.6%p) | **BW1 download (+56.1%p)** |
| D2 단발 효과 | BW1 login +3.24%p **z=+2.09 양의 유의** | BW1 download +0.03%p **z≈0 무효** |
| JA1 효과 | 무효 | 무효 (download도 무효, +0.06%p) |
| **최종 결론** | "D2 user_type customize가 진성고객 driver" | **"메일은 로그인 단계만 끌어올림. 디자인 다운로드(M1과 더 강한 상관)까진 못 만듦. 메일이 진성고객 직접 driver 아님."** |

→ v1은 로그인 retention proxy만 봐서 D2 양 신호 잡았는데, **더 강한 진성고객 신호(download)로 보면 메일이 그 단계까지 못 끌어올림**. 정직 결론.

## 1. KPI·선행지표 framework

### 1.1 팀 KPI (사용자 정의)
- **북극성**: M1 retention (가입 +30~59일 재로그인)
- **팀 존속 KPI**: 리테인 AU (저번달·이번달 모두 로그인 유저, `gold.miridih_analytics.mican_au_ex_miricle_agg.retain`)

### 1.2 선행지표 (W0/W1/BW1) — 일본 가입자 cohort 59,550명 검증

| Proxy | n_양 | P(M1\|양) | P(M1\|음) | lift | M1 설명력 |
|---|---:|---:|---:|---:|---:|
| W0 login (D+1~6) | 7,470 | 21.7% | 2.6% | +19.0%p | 54.2% |
| **W0 download (D0~6)** | **10,631** | 14.1% | 3.1% | +11.0%p | **50.1%** |
| D0 download (signup day) | 9,485 | 12.3% | 3.6% | +8.6%p | 39.0% |
| W1 login (D+7~13) | 3,531 | 35.9% | 3.1% | +32.8%p | 42.4% |
| **W1 download** | 1,273 | **48.2%** | 4.1% | **+44.2%p** | 20.6% |
| **BW1 login** (D+14~27) | 3,344 | **45.3%** | 2.6% | **+42.6%p** | 50.7% |
| **BW1 download** | **1,265** | **59.9%** | 3.8% | **+56.1%p** | 25.4% |

**핵심**:
- **다운로드 신호가 로그인 신호보다 강함** (W1·BW1 시점):
  - W1 download +44.2%p > W1 login +32.8%p
  - BW1 download **+56.1%p** > BW1 login +42.6%p
- **BW1 download가 M1 최강 단일 예측인자** — P(M1 | BW1 download) = 60%
- **D0 download 가설 검증**: 사용자가 들었던 "일각의 의견" (D0 download이 장기 retention 상관)은 **부분만 맞음**. D0 download도 lift +8.6%p로 의미 있지만 W1·BW1 download의 약 1/5~1/6. 신호 시점이 D0보다 W1·BW1 이상이 best.
- **시점별 권장**:
  - 빠른 1주차 평가 → W0 login (n 큼)
  - 중간 평가 → W1 login·download
  - 가장 강한 평가 → **BW1 download** (M1과 최강 상관)

## 2. 일본 5개 메일 캠페인 효과 — 로그인 vs 다운로드 비교

### 2.1 로그인 retention 효과 (v1 결과)

| 캠페인 | BW1 login lift | z | 결론 |
|---|---:|---:|---|
| JA1 | -0.19%p | -0.30 | 무효 |
| D1 | -0.73%p | -0.33 | 무효 |
| **D2** | **+3.24%p** | **+2.09** | **양의 유의** |
| JAS 8통 | +1.93%p | +0.41 | 약한 양 |
| EDU 3통 | (BW1 측정 cohort 부족) | — | — |

### 2.2 다운로드 retention 효과 (v2 새 결과)

| 캠페인 | D0 dl lift | W0 dl lift | W1 dl lift | BW1 dl lift |
|---|---:|---:|---:|---:|
| JA1 | +0.66%p (z≈+0.80) | -0.06 | +0.03 | **+0.06%p (z=+0.22)** |
| D1 | +0.31 | -0.29 | **+0.29%p (z≈+1.5)** | (cohort 없음) |
| D2 | -0.12 | -0.18 | +0.03 | (cohort 없음) |
| JAS | +0.75 | +1.40 | -0.29 | (cohort 없음) |
| EDU | -0.68 | -5.16 | +1.0 | (cohort 없음) |

### 2.3 로그인-다운로드 갭 = 메일의 본질적 한계

| 캠페인 | BW1 login z | BW1 download z |
|---|---:|---:|
| JA1 | -0.30 | **+0.22 (~무효)** |
| **D2** | **+2.09 (유의)** | (다른 캠페인은 BW1 download cohort 없음) |

→ **D2조차도 BW1 download는 측정 cohort 부족으로 직접 비교 불가**. JA1만 가능했고 무효.

JA1 비교: BW1 login 무효 + BW1 download 무효. **로그인 메일이 어디까지 끌어올리는가가 명확하지 않은 상태**.

해석: **메일은 funnel의 첫 단계(로그인)만 끌어올림. 다운로드(깊은 engagement)까진 메일이 못 만듦**. 진짜 진성고객 신호인 download 측면에선 5개 캠페인 모두 marginal value.

가능 mechanism:
- 다운로드 완료는 디자인 작업 완성을 의미 — **사용자 본질 motivation + UX·콘텐츠 수준**이 결정. 메일은 첫 트리거만.
- 메일로 들어온 사용자가 디자인 시작은 하지만 완성·다운로드까진 안 감 (이탈 또는 미완)
- 진짜 진성고객 = **자력 다운로드까지 도달한 사용자** — 메일이 잘 안 만듦

## 3. 다음 시즌 권장 — v2 재조정

### 3.1 v1 권장 → v2 재조정

| v1 권장 | v2 정정 |
|---|---|
| D2 user_type customize 메인 채널 | **D2 customize 효과는 BW1 login에 한정. BW1 download엔 약함. 운영 비용 대비 marginal value** |
| JAS 시퀀스 폐기 | 유지 (download 측면에서도 효과 약함 확인) |
| JA1·D1·EDU 폐기 | 유지 (download 측면에서도 무효) |
| NULL audience 발송 보류 | 유지 (음 효과 확인) |

### 3.2 메일 외 lever 권장 (더 큰 ROI)

진성고객 전환의 진짜 driver는 메일 아님 → 다음 lever:

1. **In-app 온보딩 UX 강화**: 가입 직후 D0~W0 시점 다운로드 완료까지 funnel 끊김 mechanism 분해. 어디서 이탈하는지 식별 (W0 download +11%p lift, D0+W0에 도달한 사용자가 M1의 50% 차지)

2. **디자인 추천 personalization**: user_type별 즉시 시도 가능한 template (STUDENT엔 가벼운 디자인, COMPANY엔 업무 자료)

3. **NULL audience 분해**: 가입자의 ~30% (18,288명/audience 전체)가 user_type 미설문. 모든 메일·기능에 baseline 미달. user_type 식별 UX 우선순위 ↑

4. **D2 customize는 메일 채널 안에서만 권장**: 메일 발송 자체를 결정한 경우, user_type별 cell match가 baseline 대비 BW1 login 끌어올림. 단 진성고객 직접 효과는 marginal.

### 3.3 실험 vs 확정

**확정 (실험 불필요)**:
- 메일이 진성고객 직접 driver가 아니라는 결론 (download 측면)
- JA1·D1·EDU 폐기
- NULL audience 발송 보류

**실험 필요**:
- A. NULL audience 분해 (왜 baseline 미달)
- B. In-app 온보딩 UX 개선 효과 측정 (메일 외 lever)
- C. D2 customize의 BW1 login 효과 재검증 (운영 비용 대비 가치)
- D. M1 직접 측정 (가입자 cohort가 분석 시점 +59일 도달하면)

## 4. 분석의 학습 — paradigm 변경 두 번째

| 단계 | KPI metric | 주요 결론 |
|---|---|---|
| 초기 (v3, 6_폴더) | v7 cell-level (Stickiness·LPU 등) | JAS 시퀀스 dominant |
| v1 (7_폴더) | login retention (W0·W1·BW1) | D2 단발 dominant |
| **v2 (지금)** | **+ download retention** | **메일은 funnel 첫 단계만. 진성고객 직접 driver 아님. download까지 끌어올릴 lever 별도 필요** |

→ **CRM 효과 정직 평가에는 funnel 단계별 metric 동시 필요**:
- 활성 빈도 (LPU·LDeep) — 메일 직접 클릭 효과
- 로그인 retention (W1·BW1) — 메일로 다시 들어왔나
- **다운로드 retention (W1·BW1 dl)** — 진짜 완성·진성고객 신호
- **M1 retention** — 최종 KPI (직접 측정 가능 시점 도달 시)

단일 metric은 위험. 단계별 funnel 평가 필수.

## 6. 한계·Caveat

- **BW1 download cohort 매우 좁음**: signup ≤ 04-20만 측정 가능. JA1 (8108) audience만 안정 측정. D1·D2·JAS·EDU는 download BW1 측정 거의 불가
- **D2의 BW1 login 양 신호가 download로 안 이어지는지 직접 검증 불가**: D2 BW1 download cohort 부족. 가설로만
- **다운로드 정의는 완성된 작업물 export** — 단순 "디자인 시작" 또는 "편집"보다 깊은 신호. 따라서 메일 효과 미미한 게 본질적 (메일이 깊은 motivation 안 만듦)
- **편집 metric 미포함**: 에디터 1.0/2.0 카운트 방식 상이로 제외 (사용자 명시)
- **M1 직접 측정 시점 도달 시 재검증 필요**: 본 분석은 W·BW proxy. M1 직접 측정 cohort가 누적되면 모든 결론 직접 검증 가능

## 7. 결론 한 문장

**"진성고객 전환(M1 retention)의 가장 강한 단일 예측인자는 BW1 다운로드(가입 +14~27일 디자인 다운로드, P(M1|양)=60%)이며, 일본 5개 메일 캠페인은 로그인 단계까진 일부 끌어올렸으나 다운로드 단계까진 거의 못 끌어올렸기에, 메일을 진성고객 직접 driver로 위치시키지 말고 in-app 온보딩 UX·디자인 추천·NULL audience 분해 같은 메일 외 lever에 자원을 집중하는 것이 ROI 더 큼."**
