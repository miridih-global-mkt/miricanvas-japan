## 1. 레이아웃 유형 분류 (업계 표준 기준)

이메일 레이아웃은 콘텐츠 목적과 디바이스 환경에 따라 7가지 유형으로 분류된다. 각 유형의 특성과 MiriCanvas Japan 적용 적합도를 함께 정리한다.

| 레이아웃 유형 | 구조 설명 | 모바일 적합도 | 추천 이메일 유형 | MiriCanvas 적용 |
|---|---|---|---|---|
| **단일 컬럼(Single-Column)** | 전체 콘텐츠를 1열 수직 배치 | ★★★★★ | 온보딩, 웰컴, 공지 | 기능 소개, 온보딩 |
| **2컬럼(Two-Column)** | 좌우 2열 분리 구성 | ★★★☆☆ | 제품 쇼케이스, 비교 | 플랜 비교, 기능 나열 |
| **하이브리드(Hybrid)** | 상단 1열 + 하단 2열 혼합 | ★★★★☆ | 프로모션, 제품 런칭 | 이벤트, 시즌 캠페인 |
| **역피라미드(Inverted Pyramid)** | 주목도 높은 요소 → 세부정보 → CTA | ★★★★★ | 온보딩, 공지, 웰컴 | 온보딩 시리즈 전반 |
| **지그재그(Zig-Zag)** | 이미지-텍스트 교차 배치 | ★★★☆☆ | 기능 소개, 뉴스레터 | 다기능 소개 이메일 |
| **F-패턴** | 상단 강조 → 좌측 세부내용 → CTA | ★★★☆☆ | 교육 콘텐츠, 가이드 | 튜토리얼 이메일 |
| **텍스트 위주(Alt-Text)** | 최소한의 시각 요소, 텍스트 중심 | ★★★★★ | 트랜잭션, 개인 발송 | 행동 촉구 개인 메시지 |

**MiriCanvas Japan 핵심 권장**: 단일 컬럼 + 역피라미드 조합이 일본 사용자 모바일 환경(80% 이상 모바일 오픈)에 가장 적합하다.

## 2. 인지심리학 관점에서의 이메일 구조

### 2-1. 시선 스캔 패턴

**F-패턴 (데스크톱 환경)**
- 사용자는 상단을 좌→우로 훑고, 다시 좌측을 따라 아래로 이동한다
- 핵심 정보는 상단 좌측 배치, 제목/로고/헤드라인이 이 영역에 위치해야 한다
- 텍스트량이 많은 뉴스레터, 교육용 이메일에 적합하다

**Z-패턴 (모바일 및 이미지 중심 이메일)**
- 상단 좌→우 → 대각선 아래 → 하단 좌→우 순서로 시선 이동
- CTA 버튼은 Z-패턴의 마지막 지점(우하단)에 배치 시 클릭률 향상
- 카피가 적고 비주얼 중심인 이벤트·프로모션 이메일에 유리하다

### 2-2. 핵심 인지심리학 원칙 (이메일 적용)

**사전 주의 처리(Pre-attentive Processing) — 200ms 법칙**
- 뇌는 색상, 크기, 위치를 0.2초 만에 처리한다
- CTA 버튼은 대비색(브랜드 주색과 반대색)으로 즉시 눈에 띄어야 한다
- MiriCanvas Japan 적용: 버튼 색상은 배경과 최소 4.5:1 명도 대비 확보

**폰 레스토프 효과(Von Restorff Effect) — 고립 효과**
- 주변과 다른 요소가 가장 기억에 남는다
- 주요 CTA는 형태·색상·크기를 차별화하고, 보조 링크는 텍스트로만 처리
- 하나의 이메일에 CTA 버튼을 1~2개로 제한하는 이유

**인지 부하 이론(Cognitive Load Theory)**
- 뇌는 동시에 3~5개 항목만 처리 가능
- 콘텐츠 블록 수를 3개 이하로 제한하고, 넉넉한 여백(padding 20px 이상) 확보
- 텍스트를 짧은 단락과 불릿포인트로 분리해야 한다

**계열 위치 효과(Serial Position Effect)**
- 첫 번째와 마지막 항목이 가장 잘 기억된다
- 프리헤더(preheader)에 핵심 메시지, 이메일 끝에 CTA를 다시 배치한다

## 3. 이메일 유형별 구조적 차이

### 3-1. 온보딩(Onboarding) 이메일
**목적**: 신규 사용자가 첫 핵심 가치를 경험하도록 안내

**권장 구조**:
```
[헤더: 브랜드 로고]
[환영 헤드라인 — "MiriCanvas에 오신 걸 환영합니다!"]
[핵심 액션 1개 — 번호 매긴 단계 또는 단일 CTA]
[스크린샷/GIF — 실제 제품 화면]
[CTA 버튼 — "지금 시작하기"]
[푸터: 지원 링크, 수신거부]
```

**특징**:
- 이메일 1통 = 액션 1개 원칙 (복수 CTA 금지)
- 4~7통 시퀀스, 7~10일에 걸쳐 발송
- 진행률 표시("설정 20% 완료") 포함 시 참여율 향상
- 단일 컬럼 + 역피라미드 레이아웃 권장

### 3-2. 기능 소개(Feature Announcement) 이메일
**목적**: 기존 사용자에게 신기능 인지 및 사용 유도

**권장 구조**:
```
[헤더: 로고]
[기능명 헤드라인 — 혜택 중심 표현]
[기능 설명 — 1~2문단, "무엇을 할 수 있는가" 중심]
[GIF 또는 스크린샷 — 기능 시각화]
[CTA — "새 기능 사용해보기"]
[보조 링크 — 블로그/가이드]
[푸터]
```

**특징**:
- "무엇을 만들었나"가 아닌 "사용자에게 무슨 이점이 있나" 중심 카피
- 지그재그 레이아웃 활용 시 다기능 소개에 효과적
- GIF는 1MB 이하 유지 (Gmail 클리핑 방지)

### 3-3. 이벤트/웨비나 안내 이메일
**목적**: 참가 등록 및 이벤트 인지 확산

**권장 구조**:
```
[헤더: 이벤트 브랜딩 이미지]
[이벤트 제목 — 명확하고 혜택 중심]
[날짜/시간/시간대 — 굵은 텍스트 강조]
[이벤트 소개 — 3불릿 포인트로 압축]
[발표자 소개 — 사진+직함]
[CTA — "지금 등록하기" (두드러진 버튼)]
[추가 정보: 진행 시간, 비용 무료 여부]
[푸터]
```

**특징**:
- 화요일 발송 시 등록률 21% 높음 (업계 데이터)
- 이벤트 2주 전 첫 발송, 1주 전 리마인더 발송
- CTA 버튼 2회 배치 — 상단(즉시 의향 포착)과 하단(상세 읽은 후)

### 3-4. 튜토리얼 이메일
**목적**: 특정 기능의 사용 방법을 단계적으로 안내

**권장 구조**:
```
[헤더: 로고]
[헤드라인 — "5분 만에 마스터하는 [기능명]"]
[Step 1 — 아이콘 + 설명 + 스크린샷]
[Step 2 — 아이콘 + 설명 + 스크린샷]
[Step 3 — 아이콘 + 설명 + 스크린샷]
[CTA — "직접 해보기"]
[도움 링크 — FAQ, 고객지원]
[푸터]
```

**특징**:
- 번호 매긴 단계(Numbered Steps)로 진행 상황 명확화
- 각 단계에 실제 스크린샷 삽입 — 추상성 제거
- F-패턴 레이아웃 적합

## 4. CTA 배치, 계층구조, 텍스트-이미지 비율 원칙

### 4-1. CTA(Call-to-Action) 설계 원칙

| 항목 | 권장 기준 |
|---|---|
| 버튼 최소 크기 | 44×44px (iOS) / 48×48px (Android) |
| 버튼 텍스트 | 동사+명사 구성 — "템플릿 만들기", "기능 확인하기" |
| 버튼 색상 | 배경과 4.5:1 이상 명도 대비 |
| 이메일당 CTA 수 | 주 CTA 1개, 보조 텍스트 링크 1~2개 |
| 주 CTA 위치 | 스크롤 없이 보이는 영역(Above the fold) |
| 60-30-10 색상 규칙 | 중성색 60% + 브랜드색 30% + CTA 강조색 10% |

### 4-2. 시각적 계층구조 (Typography Hierarchy)

- **헤딩 크기 비율**: 인접한 계층 간 1.3배 이상 차이
  - H1: 28~32px / H2: 22~24px / 본문: 14~16px
- **줄 간격**: 본문 1.5배 이상 (일본어 텍스트는 1.7배 권장)
- **굵기(Weight)**: 색상이 아닌 굵기로 계층 표현 (색각 이상자 접근성 고려)

### 4-3. 텍스트-이미지 비율

- **권장 비율**: 텍스트 60% : 이미지 40%
- 이미지 100% 이메일은 스팸 필터에 걸릴 위험 있음
- 모든 이미지에 alt 텍스트 필수 (이미지 비표시 환경 대비)
- 단일 이미지 파일 크기: 600px 너비, 1MB 이하

### 4-4. 이메일 너비 표준

- **최대 너비**: 600px (데스크톱/모바일 공통 표준)
- **모바일 기준**: 375px~414px (iPhone 표준 뷰포트)
- 600px 초과 시 모바일에서 가로 스크롤 발생

## 5. 모바일 최적화 원칙

### 모바일 이메일 필수 체크리스트

| 항목 | 기준값 | 이유 |
|---|---|---|
| 이메일 최대 너비 | 600px | 대부분의 이메일 클라이언트 기준 |
| 본문 폰트 크기 | 16px 이상 | 모바일 가독성 — 14px 미만 자동 확대 발생 |
| 버튼 최소 크기 | 44×44px | iOS HIG 기준 터치 타깃 |
| 이미지 alt 텍스트 | 필수 | iOS 메일 앱 기본 이미지 비표시 |
| 단일 컬럼 레이아웃 | 권장 | 2컬럼은 모바일에서 깨질 수 있음 |
| 패딩(여백) | 상하 20px / 좌우 16px | 엄지 조작 영역 확보 |
| 링크 간격 | 최소 8px | 오탭 방지 |

### 일본 시장 특수 고려사항

- 일본 사용자의 **80% 이상**이 모바일에서 이메일 열람
- iOS(iPhone) 점유율이 압도적 — iOS 메일 앱 기준 최우선 테스트
- 일본 통신사(DoCoMo, SoftBank) 메일 클라이언트는 이미지 자동 차단 설정이 많아 **텍스트 위주 레이아웃**이 유리
- 한글·영문보다 일본어(히라가나·카타카나·한자 혼합)는 자간 처리에 주의 — `word-break: break-all` CSS 적용 권장

## 핵심 요약 (MiriCanvas Japan 적용 관점)

1. **레이아웃**: 단일 컬럼 + 역피라미드가 기본. 기능 소개에는 지그재그, 튜토리얼에는 F-패턴 활용
2. **인지심리학**: 이메일 1통당 메시지 1개, CTA 1개 원칙 준수. 색상 대비로 CTA 즉시 식별 가능하게
3. **모바일**: 600px 너비, 16px 폰트, 44px 버튼 — 일본 사용자 80% 이상이 모바일 오픈
4. **계층구조**: H1 28px / H2 22px / 본문 16px, 일본어 행간 1.7배
5. **이미지**: 텍스트 60:이미지 40 비율, 이미지당 1MB 이하, alt 텍스트 필수
