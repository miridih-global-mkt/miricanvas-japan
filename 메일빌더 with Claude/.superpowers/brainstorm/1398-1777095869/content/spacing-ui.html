C. 프리셋만

5단계 정도 프리셋만 노출, 정밀 조정 불가
장점: UI 가장 단순, 일관성 높음
단점: 사용자가 "딱 맞는 값"이 안 나올 수 있음. 자유도 낮음

💡 추천: A (프리셋 + 맞춤 펼치기)
대부분의 경우 "보통" 같은 프리셋 한 번 클릭으로 끝. 진짜 미세 조정이 필요할 때만 "맞춤" 펼쳐서 4방향 숫자 입력. UI는 평소엔 깔끔하고, 필요할 때만 복잡해짐.

적용 범위

  • 섹션: 4방향 패딩 (위/아래/왼/오른쪽) — <td>의 padding
  • 모듈: 위/아래 마진 (모듈 간 간격 조절용) — 좌우는 보통 부모 섹션이 결정
  • 저장 형식: 사용자가 디폴트와 다르게 바꿨을 때만 JSON에 padding 필드 명시. 디폴트면 생략 → 카탈로그 원본과 100% 동일하게 렌더
