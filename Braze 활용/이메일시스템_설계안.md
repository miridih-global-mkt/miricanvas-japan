## 콘텐츠 블록

### CB-1: 헤더블록
- 로고 이미지(120px) 중앙 정렬
- 하단 구분선 `border-bottom: 1px solid #eeeeee`
- H1 타이틀은 각 템플릿 본문 첫 `<tr>`에 직접 포함 (Liquid 미사용으로 블록 내 가변 요소 불가)

### CB-2: 푸터블록
- 다크 행(`#333333`): 회사명 / HP URL(`www.miricanvas.com/ja`) / 문의처 / 주소
- 라이트 행(`#f4f4f4`): 수신거부 링크 (`{{${set_user_to_unsubscribed_url}}}`)
- 두 `<tr>`를 하나의 블록으로 묶음

## 이메일 템플릿 4유형

### T-A: Feature Spotlight（기능 스포트라이트）
대상: D3/D5 기능 소개 이메일 (배경제거, 프레임, 템플릿믹스, 북마크, 폰트)

```
{{CB-1 헤더블록}}
[H1 타이틀]
──────────
[도입 텍스트 2-3줄]
[GIF 또는 스크린샷]
[사용법 하이라이트 박스]  ← #e9f9fb + 좌측 border #26c7d9
[CTA 버튼]  ← 1개
[보너스카드 섹션 ← 선택]  ← #e9f9fb 배경, 흰색 내부카드
{{CB-2 푸터블록}}
```

### T-B: Step-by-Step Guide（단계별 가이드）

```

──────────
[도입 텍스트]

[스텝 헤더 박스]  ← 하이라이트 박스

[GIF/이미지]
[스텝별 CTA]  ← 스텝 당 1개까지

↑ N스텝 반복 ↑

[보너스카드 섹션 ← 선택]
{{CB-2 푸터블록}}
```

### T-C: Rich Content（멀티섹션 심화）
대상: 리뷰, AIP 딥다이브, 친구소개(플랜 비교), 복수 기능 묶음

```
{{CB-1 헤더블록}}
[H1 타이틀]
──────────
[섹션 타이틀 바]  ← #e9f9fb 배경 + border-bottom
[하이라이트 박스 + 텍스트/인용/비교표] × N
[중간 CTA ← 선택]
[추가 하이라이트 + 이미지] × M
[하단 CTA]
{{CB-2 푸터블록}}
```

비교표(Free vs Pro) 컴포넌트는 이 유형 내 선택 요소로 처리.

### T-D: Announcement/Event（이벤트/공지）
대상: 웨비나, 캠페인, 이벤트 고지

```

──────────
[이벤트 타이틀 바]  ← #e9f9fb 배경
[도입 텍스트]
[이벤트 상세 하이라이트 박스]  ← 날짜/장소/출연자 등
[당일 내용 목록]
[CTA 버튼]  ← 1개
[보조 섹션 ← 선택]
{{CB-2 푸터블록}}
```

## 디자인 방침

### 유지
- 600px 싱글 컬럼
- 테이블 레이아웃 (MSO/Outlook 호환)
- 색상: `#26c7d9` 메인 / `#e9f9fb` 배경 / `#333333` 텍스트
- GIF 애니메이션 (기능 시연)
- 일본어 경어체 (です・ます調)
- 한국어 버전 스타일 미도입 (500px / 배경 이미지 / `<button>` 요소)

### 수정 (기존 버그)
- `lang="ko"` → `lang="ja"` (일본 대상 파일 전체)
- CSS 오타: `mso-table-lspace` 중복 → 두 번째를 `mso-table-rspace`로
- CTA 버튼 패딩: `14px` → `16px` 상하 (최소 44px 터치 타깃)
- 푸터 HP URL: `www.miricanvas.com` → `www.miricanvas.com/ja` 통일

### 개선 (외부 연구 반영)
- 본문 텍스트 압축: 단락당 3줄 이내, 1메일 1테마
- 이미지 alt 텍스트 보완 (텍스트 전용 환경 대응)
- CTA 개수 절제: T-A·D 최대 1개, T-B 스텝당 1개(최대 3개), T-C 최대 2개(중간+하단)
