## 1. 개요 / 목적

「ミーキャンPPTで私の○○自慢」 — 미리캔버스 일본판 사용자를 대상으로 한 **시리즈형 UGC 콘테스트** 사이트.
사용자가 미캔에서 만든 PPT 링크를 응모 → 다른 사용자가 좋아요(투표) → 순위 → 상위에게 리워드.

**기대 효과**

- 응모자가 자기 SNS에 작품 페이지를 공유 → 외부 유입 / 미캔재팬 브랜드 노출
- 시즌제 운영으로 트래픽 반복 점화
- "다른 사람도 미캔으로 이런 걸 만든다"는 사회적 증거 축적

**Non-goals (이번 범위 아님)**

- 정식 회원 시스템 / 마이페이지
- 결제 / 상금 자동 지급
- 응모작 편집 기능 (어디까지나 미캔 결과물 링크만 받음)

## 2. 타깃 / 핵심 시나리오

- **타깃**: 일본 거주 미리캔버스 사용자. UI 언어 **일본어 only**.
- **시나리오**
  1. 응모자: 미캔에서 PPT 제작 → 링크 복사 → 응모 폼 → 메일 확인 → 응모 확정 → SNS 공유
  2. 투표자: SNS/검색에서 진입 → 마음에 드는 작품 좋아요 (간단 인증 1회) → 다른 작품에도 좋아요
  3. 응모자 재방문: `自分の作品をさがす`로 자기 응모작 찾아 현재 좋아요/순위 확인

## 4. 메인 UI: 갤러리 + 옵션 지도

### 결론: **갤러리가 기본 골격, 지리형 시리즈에만 [ギャラリー | マップ] 토글**

- 모든 시리즈 공통: 갤러리 카드 그리드
- `series.geo_enabled = true`인 시리즈만 지도 보기 토글 노출
- 지도 보기: 일본 도도부현 GeoJSON + 핀 → 핀 클릭 시 해당 지역 응모작 팝오버

**카드 구성 (필수)**

```
┌────────────────────────┐
│  썸네일 (16:10, OG)     │
├────────────────────────┤
│  제목                   │
│  by 닉네임 또는 匿名     │
│ ──────────────────     │
│  ♥ 124      📅 5/18    │
└────────────────────────┘
```

→ **좋아요 수는 카드에 항상 노출** (사용자가 카드 단위로 인기도를 한눈에 보기 위함).

**정렬**: `[人気順 ▼]` / `[新着順]` 토글. default = 人気順.

## 5. 랭킹 (페이지 상단 노출)

### 요구사항
- 페이지 상단(히어로 바로 아래)에 항상 노출
- 탭 2개: **累計ランキング** / **今月のランキング**
- 상위 3위는 포디움 카드 (🥇🥈🥉 + 썸네일 + 제목 + 작가 + 좋아요수)
- 4~10위는 컴팩트 리스트
- 상위 입상자에게 리워드 지급 → 랭킹은 사이트의 핵심 동기 부여 요소

### 누적 vs 이번달
| 구분 | 범위 | 리워드 |
|---|---|---|
| **累計** | 현재 시리즈 전 기간 누적 좋아요수 | 시리즈 종료 시 최종 1·2·3등 / 그랑프리 |
| **今月** | 매월 1일~말일 그 달에 받은 좋아요수 | 월간 베스트 — 매월 리워드 |

→ 데이터 모델에서 `vote.created_at`이 있으면 양쪽 다 계산 가능.

## 6. 응모 (Submit) 플로우

### 폼 필드

| 항목 | 필수 | 검증 |
|---|---|---|
| 미캔 링크 (URL) | ✅ | `miricanvas.com/v2/ja/design2/v/` 패턴 검증. OG 미리보기 즉시 표시 |
| 작품 제목 | ✅ | 1~40자 |
| 이메일 | ✅ | 응모자 본인 인증 + **`内 作品 찾기`에도 사용** |
| 표시 이름 (닉네임) | 선택 | 빈칸이면 "匿名" |
| 설명 | 선택 | 200자 이내 |
| 도도부현 | (지리 시리즈만) ✅ | 47개 select |
| 규약 동의 | ✅ | 초상권/저작권 본인 책임 명시 |

### 본인 인증

- 폼 제출 → 입력 이메일로 매직링크 → 클릭 시 응모 확정
- 미확인 응모는 24시간 후 자동 삭제
- 같은 이메일은 같은 시리즈에 **최대 3개**까지 응모 가능

### 썸네일

- 1차: 미캔 링크의 `og:image` 자동 추출 후 자체 스토리지 캐시
- 실패 시: 기본 더미 + 응모자에게 안내 메일

## 7. 좋아요 / 투표 시스템

### 규칙
- **1명이 같은 작품에 1번만** 좋아요 가능
- **1명이 여러 작품에는 다 좋아요 가능** (= "Like" 모델, 양자택일 X)
- 회원가입 절차는 만들고 싶지 않음

### 채택안: 2단 방어선

1. **1차 (가벼움)**: 첫 좋아요 시 LocalStorage 토큰 발급 + 서버 측 IP rate limit
2. **2차 (확실)**: Google / X(Twitter) / LINE **OAuth 1-클릭 로그인** — "회원가입" 단어 노출 X, "ログインして応援" 같은 표현 사용

OAuth 식별자 단위로 `vote.voter_id` 저장 → 같은 식별자가 같은 작품에 두 번 누르면 무시.

### UX
- 카드/상세 페이지에 실시간 좋아요 수 표시
- 내가 누른 작품에는 채워진 ♥
- 누른 직후 SNS 공유 모달(선택): "○○に応援しました!"

## 8. 내 작품 찾기 (Find My Work)

응모자가 자기 응모작을 빠르게 찾을 수 있는 기능. SNS 공유용 URL을 얻거나 현재 랭킹을 확인하기 위함.

### UX
- 헤더 / 히어로의 `自分の作品をさがす` 버튼 → 모달
- 이메일 입력 → "確認する"
- 결과:
  - 해당 이메일로 응모한 작품 리스트 (시리즈 무관 전부) 카드 형태로 노출
  - 각 카드에 "現在の順位 / 좋아요수 / 작품 페이지 링크 / 共有する"
- 이메일 매칭은 응모 시 입력한 이메일 기준. 추가 인증 없이 조회만 허용 (URL은 어차피 공개 페이지)

---

## 9. SNS 공유 (필수)

응모자가 자기 작품을 SNS에 올려야 좋아요가 늘기 때문에 마찰 최소화.

- 응모 완료 화면 / 작품 상세 페이지에 **자동 트윗 (X intent URL)** 버튼
- LINE / X / Facebook / URL 복사 버튼
- 작품별 동적 OG 이미지(`/api/og`): 썸네일 + 제목 + 좋아요수가 들어간 카드
- LINE 공유는 일본 시장 필수

## 10. 데이터 모델 (대략)

```
series
  id, slug, title_ja, description_ja, hero_image,
  starts_at, ends_at, status (draft/live/voting_closed/archived),
  geo_enabled (bool), display_order

submission
  id, series_id, mican_url, title, description,
  display_name (nullable), submitter_email,
  prefecture (nullable), thumbnail_url,
  status (pending_email/active/rejected),
  vote_count_total (denormalized),
  vote_count_this_month (denormalized, 일별 갱신 job),
  created_at, confirmed_at

vote
  id, submission_id,
  voter_id (OAuth sub or hashed local token),
  voter_provider (google/x/line/anon),
  ip_hash, created_at
  UNIQUE(submission_id, voter_id)

flag
  id, submission_id, reason, reporter_ip_hash, created_at
```

랭킹 계산:

- 累計: `ORDER BY vote_count_total DESC`
- 今月: `ORDER BY vote_count_this_month DESC`
- 월 바뀌면 cron으로 `vote_count_this_month` 리셋 + 전월 베스트 스냅샷 보관

## 15. 열린 이슈 (의사결정 필요)

1. **OAuth 프로바이더 우선순위**: Google / X / LINE 중 어디까지 — LINE은 일본 보급률 매우 높지만 심사 절차 있음
2. **리워드 내용**: 누적 1·2·3위 / 월간 베스트 각각 무엇을 주는가
3. **종료 후 응모작 보존**: 영구 노출 vs 응모자 삭제 요청 가능 (GDPR 관점)
4. **응모 후 수정 허용**: 제목/설명만 수정? 링크 변경은 불가 추천
5. **미성년자 응모 정책**: 일본 보호자 동의 이슈 → 약관 명시 필요
6. **누적 랭킹 범위**: 현재 시리즈 안 누적 vs 시리즈 가로지른 전체 누적 — 본 가이드는 *시리즈 내 누적*으로 가정
