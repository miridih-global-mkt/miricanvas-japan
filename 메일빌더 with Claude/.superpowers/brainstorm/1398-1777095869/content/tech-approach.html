어떤 기술로 만들까요?

비개발자 관점에서 차이가 큰 3가지 옵션. 추천 표시(⭐) 있음.

A

⭐ 단순 — Flask + 바닐라 JS + CDN 라이브러리 (추천)

구성: 백엔드는 지금처럼 Flask (Python). 프론트엔드는 HTML+CSS+JS 파일 몇 개. 추가 빌드 도구 없음.

외부 라이브러리(CDN 한 줄): 드래그앤드롭(Sortable.js), 리치 텍스트(Quill)

장점

  • npm/Node 등 추가 도구 불필요 — 지금 환경 그대로
  • python app.py 한 줄로 즉시 실행
  • Claude Code가 안정적으로 코드 수정 가능 ("X 부분 바꿔줘"가 쉬움)
  • 비개발자도 코드 흐름 따라가기 가장 쉬움

단점

  • 기능 늘어나면 상태 관리 코드가 약간 길어짐
  • 단, 단일 사용자·단일 페이지라 큰 문제 안 됨
B

가벼운 프레임워크 — Flask + Alpine.js

구성: 백엔드 Flask, 프론트엔드는 Alpine.js (HTML 안에 마크업으로 동작 정의). 빌드 도구 없음.

장점

  • 상태 관리가 깔끔
  • 빌드 도구 없음 (A와 동일하게 단순)

단점

  • Alpine 문법(x-data, @click)을 사용자가 보게 됨 — 약간의 학습/혼란
  • 업계 점유율 낮아서 Claude의 학습량도 A보다 적음
C

본격 SPA — React/Vue + Flask API

구성: 프론트엔드는 React 별도 앱 (npm 빌드), 백엔드 Flask는 JSON API 제공.

장점

  • 가장 강력 — 큰 기능 확장 시 유리
  • 업계 표준

단점

  • npm 빌드 도구 필요 → 환경 복잡도↑
  • 실행 단계가 2개로 늘어남 (백엔드 + 프론트엔드 dev 서버)
  • 비개발자가 코드를 직접 봐야 할 때 진입장벽 큼
  • 로컬 단일 사용자 도구엔 과한 선택

📌 데이터 모델은 셋 다 동일: 메일은 JSON 트리(섹션→모듈)로 저장 → 600px 이메일 HTML로 렌더링. 드래그앤드롭은 배열 순서 변경, 코드 보기는 트리→HTML 변환 결과.
