작성일: 2026-05-21  
작성 근거: 전체 HTML 파일 47개 (12개 폴더) 분석 + 이론/글로벌 사례/일본 트렌드 리서치

## 설계 원칙

### v1에서 달라진 점
- v1: 4가지 고정 템플릿 (T-A~T-D) → v2: 모듈 라이브러리 + 10가지 템플릿 레시피
- v1: 변형 수 많은 레이아웃 위주 반영 → v2: 변형 수와 무관하게 구조적 DNA 기준 분류
- v1: module_catalog 미참조 → v2: module_catalog를 시스템의 중심으로 위치

### 설계 철학
1. **모듈 우선** — 이메일은 모듈 조합으로 만든다. 템플릿은 "레시피(어떤 모듈을 어떤 순서로)"다.
2. **1메일 1액션** — 이메일 한 통에 CTA 1개 원칙 (T-3 스텝형은 스텝당 1개까지)
3. **일본어 경어체 필수** — です・ます調, 직접 구매 유도 금지, 관계 형성 우선
4. **콘텐츠 블록 활용** — 헤더/푸터는 `mc_ja_header` / `mc_ja_footer` 콘텐츠 블록으로 고정

| `card_1col_half` | 2열 카드 (좌:썸네일 / 우:텍스트+CTA) | 리스트형 나열 |

## 템플릿 레시피 (10종)

### T-1: Feature Spotlight (기능 스포트라이트)
**개요:** 단일 기능, GIF 1개, 사용 하이라이트, CTA 1개  
**모듈 구성:**
```
mc_ja_header
H1 (좌 or 중앙 정렬)
body (도입 텍스트)
[GIF 이미지]
highlight box (こんな時に使う)
cta_solo or cta_microcopy
mc_ja_footer
```
**사례:** 배경제거, 프레임, 북마크(D5), 템플릿믹스, 에듀 시리즈  
**CTA 수:** 1개  
**비고:** 가장 많이 쓰이는 유형. 기존 T-A.

---

### T-2: Multi-Demo Feature (복수 데모 기능 소개)
**개요:** 단일 기능의 하위 항목(예: 폰트 4종류, 표 못생긴/예쁜)을 각각 시연  
**모듈 구성:**
```
mc_ja_header
H1
body (도입 + 문제 제기)
[소제목 + 이미지/GIF + 설명] × N
highlight box (핵심 팁 or 총정리)
cta_headline_body or cta_microcopy
mc_ja_footer
```
**사례:** 표.html (비포/애프터), 폰트.html (4카테고리), 북마크A/B  
**CTA 수:** 1~2개  
**비고:** T-1과 콘텐츠 주제는 같지만 "같은 기능의 여러 측면을 순서대로 보여줌"이 구조적 차이.

---

### T-3: Step-by-Step Guide (단계별 가이드)
**개요:** N개 스텝을 번호 순서대로 전개, 스텝당 GIF + CTA  
**모듈 구성:**
```
mc_ja_header
H1
body (도입)
[step_block] × N  ← 스텝당 1개 CTA까지 허용
box (보너스/AI 섹션, 선택)
mc_ja_footer
```
**사례:** 4스텝 시리즈 (D1, 온보딩시리즈)  
**CTA 수:** 스텝당 최대 1개 (최대 4~5개)  
**비고:** 기존 T-B. 스텝 수 변동 가능.

---

### T-4: Grid Showcase (그리드 쇼케이스)
**개요:** 여러 아이템(템플릿/기능)을 그리드로 펼쳐 보여주고, 주요 항목은 추가 GIF 설명  
**모듈 구성:**
```
mc_ja_header
H1
body (도입)
grid_4col (전체 아이템 썸네일)
[header + body + GIF] × N  ← 주요 항목별 심화 설명
cta_solo
mc_ja_footer
```
**사례:** 차트.html (8종 그리드 + 콤보차트 3단 설명)  
**CTA 수:** 1개  
**비고:** 에듀 class.html의 "템플릿 모음 보여주기"에도 응용 가능.

---

### T-5: Conceptual Deep Dive (개념 심화)
**개요:** 추상적 개념이나 기술 원리를 설명하고 실제 결과로 증명  
**모듈 구성:**
```
mc_ja_header
H1 (질문/호기심 유발형 제목)
body (개념 설명)
highlight box × 2 (핵심 포인트 2개)
cta_microcopy (상단 CTA)
[GIF 데모]
[결과물 이미지 × N]
cta_headline_body (하단 CTA)
mc_ja_footer
```
**사례:** AIP.html, AIP_curiosity.html  
**CTA 수:** 2개 (상단/하단)  
**비고:** 기존 T-C 일부.

---

### T-6: Social Proof (후기 스택)
**개요:** 실사용자 리뷰·후기를 여러 블록으로 나열  
**모듈 구성:**
```
mc_ja_header
H1
body (도입 — 후기 출처 소개)
[review_quote] × 3~5
cta_microcopy
mc_ja_footer
```
**사례:** 리뷰.html, 리뷰_number.html  
**CTA 수:** 1개  
**비고:** 기존 T-C 일부였으나 구조적으로 독립. 신뢰 구축 목적.

---

### T-7: Event / Webinar (이벤트/웨비나)
**개요:** 이벤트 일정·내용을 알리고 참여를 유도  
**모듈 구성:**
```
mc_ja_header
[배너 이미지] (hero 안에 포함 or 별도 tr)
H1 (이벤트 타이틀)
body (도입 + 이벤트 개요)
highlight box (날짜/장소/출연자/내용 목록)
cta_solo or cta_microcopy
[보조 섹션, 선택]
mc_ja_footer
```
**사례:** 웨비나 Keitaro 시리즈 (churn/new/remind)  
**CTA 수:** 1개  
**비고:** 기존 T-D. churn/new/remind 버전은 수신자 세그먼트만 다르고 구조 동일.

---

### T-8: Newsletter / Digest (뉴스레터/다이제스트)
**개요:** 복수의 독립 토픽을 한 메일에 묶음. 각 섹션이 별도 주제  
**모듈 구성:**
```
mc_ja_header
[배너 이미지] (히어로 안에 포함)
[header + body + box] × N  ← 섹션별 반복
mc_ja_footer
```
**사례:** 웨비나 2605 시리즈 (날짜별 뉴스레터)  
**CTA 수:** 섹션당 1개  
**비고:** 신규. T-7(단일 이벤트)과 달리 복수 주제 묶음.

---

### T-9: Referral / Comparison (친구소개/비교)
**개요:** 플랜 비교표 + 단계별 소개 프로세스  
**모듈 구성:**
```
mc_ja_header
H1
body (도입 + 혜택 설명)
comparison_table (Free vs Pro 비교)
referral_steps (1-2-3 단계 이미지 포함)
cta_headline_body
mc_ja_footer
```
**사례:** 친구소개.html  
**CTA 수:** 1~2개  
**비고:** 신규. 구조상 comparison_table이 핵심.

---

### T-10: Relationship / Soft Touch (관계 메일)
**개요:** 구매 유도 없이 관계 유지, Q&A 제공, 영상 안내  
**모듈 구성 (A타입 — Q&A형):**
```
mc_ja_header
H1 (감사 or 사과 톤)
body (정중한 본문)
qa_section (질문 + 답변 + CTA)
mc_ja_footer
```
**모듈 구성 (B타입 — 영상 가이드형):**
```
mc_ja_header
H1
body (공감 + 설명)
video_thumb × 1~2 (유튜브 썸네일)
body (마무리)
mc_ja_footer
```
**사례:** 감사A/B.html (A타입), 무료필터.html (B타입)  
**CTA 수:** 0~1개 (영상 썸네일이 CTA 역할)  
**비고:** 신규. 일본 이메일 문화의 관계 형성 중시에 최적화된 유형.
