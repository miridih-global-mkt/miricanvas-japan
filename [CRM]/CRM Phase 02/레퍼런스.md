## 섹션 1: SaaS 제품 온보딩 이메일 우수 사례

### 사례 1: Grammarly — 단일 액션 집중형 온보딩

**브랜드**: Grammarly (AI 글쓰기 보조 SaaS)
**이메일 유형**: 온보딩 #1 — 핵심 액션 유도

**레이아웃 구조**:
```
[로고 헤더]
[헤드라인: 명확한 다음 단계 제시]
[Step 1: 스크린샷 + 설명]
[Step 2: 스크린샷 + 설명]
[Step 3: 스크린샷 + 설명]
[CTA 버튼 1개: "확장 프로그램 설치하기"]
[푸터]
```

**레이아웃 특징**:
- 단일 컬럼, 번호 매긴 3단계 설치 가이드
- 각 단계마다 실제 UI 스크린샷 삽입 — 사용자가 무엇을 보게 될지 미리 확인 가능
- CTA 버튼 1개만 존재 — 집중도 최대화
- 제품 설명 없음 — "지금 당장 해야 할 일" 하나에만 집중

**효과적인 이유**:
- 첫 온보딩 이메일에서 기능 나열을 완전히 배제하고 단 하나의 설치 행동만 요청
- 스크린샷이 추상성을 제거해 "할 수 있겠다"는 자신감 부여

**MiriCanvas Japan 적용 인사이트**:
첫 온보딩 이메일은 "MiriCanvas 시작하기" 단 하나의 액션만. 사이트 내 첫 템플릿 선택 화면 스크린샷 삽입 권장.

### 사례 2: Loom — 제품으로 제품 설명하기

**브랜드**: Loom (비디오 메시지 SaaS)
**이메일 유형**: 온보딩 — 제품 시연형

**레이아웃 구조**:
```
[로고 헤더]
[개인 환영 문구]
[임베디드 비디오 썸네일 (클릭 시 실제 Loom 영상으로 연결)]
[담당자 서명 (VP of Product 직접 발송 형식)]
[다음 단계 텍스트 링크 3개]
[푸터]
```

**레이아웃 특징**:
- 이미지 최소화, 비디오 썸네일이 중심 비주얼
- 실제 Loom 영상을 이메일에 삽입 — "제품으로 제품을 설명"
- 하단 링크는 텍스트 형태 — 버튼 경쟁 없음
- 개인 발송 형식(from: 실제 직원 이름)으로 친근감 극대화

**효과적인 이유**:
- Loom은 영상 메시지 도구 → 영상으로 온보딩하는 것 자체가 제품 가치 증명
- 영상 시청 전 불안감(어렵겠지?) 제거 효과

**MiriCanvas Japan 적용 인사이트**:
"미리캔버스로 만든 실제 디자인 결과물"을 이메일 비주얼로 활용. 예: 미캔재팬 팀이 미캔버스로 만든 소셜 카드를 이메일 헤더 이미지로 삽입.

### 사례 3: Trello — 자기주도형 리소스 메뉴

**브랜드**: Trello (프로젝트 관리 SaaS)
**이메일 유형**: 웰컴 + 온보딩 — 자기주도 탐색형

**레이아웃 구조**:
```
[로고 헤더]
[따뜻한 환영 헤드라인]
[앞으로 받을 이메일 내용 예고 (기대 설정)]
[리소스 링크 4개 — 색상으로 구분]
  ├ 시작 가이드
  ├ 영상 튜토리얼
  ├ 팀 초대하기
  └ 사례 보기
[CTA 버튼: "Trello 시작하기"]
[푸터]
```

**레이아웃 특징**:
- 색상 코딩된 리소스 링크 — 사용자 유형별 자기주도 탐색 가능
- 앞으로 받을 이메일 시리즈를 미리 예고 → 수신거부율 감소
- 단일 컬럼 구조, 스크롤 최소화

**효과적인 이유**:
- 모든 사용자가 같은 단계에 있지 않음을 인식하고, 선택지 제공
- 이메일 구독 관계를 미리 정의해 신뢰 구축

**MiriCanvas Japan 적용 인사이트**:
웰컴 이메일에서 "다음 이메일에는 일본어 템플릿 모음을 보내드립니다" 등 시퀀스 예고로 오픈율 유지.

### 사례 4: Typeform — 단계별 스크린샷 가이드

**브랜드**: Typeform (인터랙티브 설문 SaaS)
**이메일 유형**: 온보딩 — 첫 폼 생성 유도

**레이아웃 구조**:
```
[로고 헤더]
[헤드라인: "첫 폼을 만들어보세요"]
[Step 1 — 아이콘 + 텍스트 + 화면 캡처]
[Step 2 — 아이콘 + 텍스트 + 화면 캡처]
[Step 3 — 아이콘 + 텍스트 + 화면 캡처]
[CTA: "폼 만들기"]
[도움이 필요하세요? 지원 링크]
[푸터]
```

**레이아웃 특징**:
- 번호 + 아이콘 + 스크린샷 3중 구조로 각 단계 명확화
- 각 스크린샷은 실제 Typeform 편집 화면 — "이렇게 생겼어요" 안내
- 마지막에 지원 링크 배치 — 마찰 발생 지점에서의 탈출구 제공

**MiriCanvas Japan 적용 인사이트**:
튜토리얼 이메일에서 "템플릿 선택 → 텍스트 편집 → 다운로드" 3단계를 스크린샷과 함께 구성.

### 사례 5: ActiveCampaign — 퀵스타트 가이드형

**브랜드**: ActiveCampaign (마케팅 자동화 SaaS)
**이메일 유형**: 온보딩 — 빠른 시작 가이드

**레이아웃 구조**:
```
[로고 헤더]
[상단 개인 로그인 링크 (이메일 저장 유인)]
[헤드라인: "5분 설정 가이드"]

  1. [액션 설명] [버튼]
  2. [액션 설명] [버튼]
  3. [액션 설명] [버튼]

**레이아웃 특징**:

- 상단의 개인 로그인 링크는 이메일 자체의 지속적 가치 부여
- 극도로 간결한 복사 — 설명 최소화

**효과적인 이유**:
- 사용자가 준비된 순간에 바로 행동 가능한 구조
- 이메일을 '저장해두고 싶은 레퍼런스'로 포지셔닝

### 사례 1: HubSpot — INBOUND 컨퍼런스 초대

**브랜드**: HubSpot
**이메일 유형**: 대형 이벤트 초대

**레이아웃 구조**:
```
[이벤트 브랜딩 헤더 이미지]
[이벤트명 + 날짜/장소 — 굵게 강조]
[이벤트 가치 제안 — 3불릿]
[키노트 발표자 — 사진 + 이름 + 직함]
[CTA: "지금 등록하기" (상단)]
[세션 미리보기 — 주요 트랙 소개]
[CTA: "자리 확보하기" (하단)]

- CTA 버튼 2회 배치 (상/하단) — 스크롤 위치 어디서든 등록 가능
- 발표자 사진 삽입으로 신뢰도와 기대감 동시 생성
- 이른 등록 혜택으로 즉시 행동 유인

**타이밍 전략**:
- 2주 전: 첫 초대 이메일
- 1주 전: 리마인더 (혜택 강조)

## MiriCanvas Japan 이메일 시스템 설계 시사점 요약

1. **온보딩**: 이메일 1통 = 액션 1개. 스크린샷으로 실제 제품 화면 보여주기. 4~7통 시퀀스 구성
2. **기능 소개**: "무엇이 추가됐나"가 아닌 "사용자가 무엇을 할 수 있게 됐나" 중심 카피. GIF나 스크린샷 필수
3. **이벤트/웨비나**: CTA 상하 2회 배치, 날짜·시간·소요시간·무료 여부 상단 명시, 2주 전 첫 발송
4. **공통**: Canva 사례처럼 MiriCanvas로 만든 실제 결과물을 이메일 비주얼로 활용해 제품 가치 자연 시연
