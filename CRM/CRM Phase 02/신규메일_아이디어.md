## 아이디어 1 — "生成 → 編集" 황금 루트 체험 유도 메일
**타겟**: NULL / UNKNOWN (가입 후 아직 아무것도 안 한 유저)  
**타이밍**: D+1  
**소구점 기반**: 백서 핵심 인사이트 — "템플릿 생성 + 편집 두 경험을 한 유저 = 황금 유저"

```
구조:
[헤더] 「MiriCanvasを使いこなす人はみな、このルートを通っています」
[2단계 플로우 카드]
  STEP 1: テンプレートを選んで生成（3分）
           → 주제 입력 → AI가 30장 완성
  STEP 2: 1つだけ自分用に変える
           → 로고 색상 or 텍스트 1개 수정
[CTA] 「今すぐ1枚を自分のものに」
[하단] 「ここまでできた人の継続率は、そうでない人の3倍です」
```

**HTML 포인트**: 2단계 플로우를 좌→우 화살표. STEP1 배경 mc-soft(민트), STEP2 배경 네이비. 여정감 강조.

## 아이디어 2 — 경쟁사 비교 카드 메일 (switcher 타겟)
**타겟**: BUSINESS / COMPANY  
**타이밍**: D+3~4 내삽  
**소구점 기반**: "Gammaから乗り換えた理由" — PPTX + 완전 편집

```
구조:
[헤더] 「Gammaと迷っているあなたへ」
[3항목 비교 카드]
  ① AIPスライド: MiriCanvas 30枚(無料) vs Gamma 10枚
  ② PPTX出力: MiriCanvas ◎ 完全対応 vs Gamma ✗ 不可
  ③ 編集自由度: MiriCanvas ◎ 1mm vs Gamma ✗ 位置変更不可
[페인포인트 카드]
  「Gammaで作って、PowerPointで開いたら崩れた経験はありませんか？」
[CTA] 「まず30枚を無料で生成してみる」
```

**HTML 포인트**: 3열 매트릭스. MiriCanvas 열 민트 컬러. ✗는 red-soft. 기능 비교 전에 페인포인트로 오픈.

## 아이디어 3 — user_type별 맞춤 템플릿 쇼케이스 메일
**타겟**: 세그먼트별 각각 버전 (STUDENT / BUSINESS / EDUCATION)  
**타이밍**: D+5~8 내삽  
**소구점 기반**: "당신을 위해 만들어진 것"

```
STUDENT 버전:
[헤더] 「学生が一番使っているテンプレート3選」
  ① レポートカバー — "提出物が一気に本格的に見える"
  ② データ比較表 — "調査データをグラフで瞬時に"
  ③ 発表スライド — "授業発表が3分で完成"
[CTA] 「このテンプレートを使う」

BUSINESS 버전:
[헤더] 「営業・提案に使われているテンプレート3選」
  ① 提案書 — "初稿3分、修正ゼロ、PPTX出力"
  ② 会社紹介 — "コーポレートカラーを自動適用"
  ③ 事例共有 — "Beforeを伝え、Afterで締める"
[CTA] 「提案書を今日作ってみる」

EDUCATION 버전:
[헤더] 「教育・指導の現場で使えるテンプレート3選」
  ① 授業スライド — "板書をそのままスライドに"
  ② 学習プリント — "Word作業をゼロに"
  ③ クラス通知 — "保護者へのお知らせを5分で"
[CTA] 「授業用テンプレートを見る」
```

**HTML 포인트**: 3열 카드 그리드. 각 카드 실제 템플릿 썸네일 + 한 줄 카피. 브레이즈 user_type 태그로 분기.

## 아이디어 4 — "AI가 이렇게 만들었다" 결과물 임팩트 메일
**타겟**: NULL / UNKNOWN (AI 기능 미경험 유저)  
**타이밍**: D+2  
**소구점 기반**: 백서 1.1 — "텍스트 1행 → 30장. 3분 이내."

```
구조:
[헤더] 「テキスト1行、3分後にどうなったか」
[Before/After]
  BEFORE 입력: 「新しいコーヒーショップのオープン案内を作って」(회색 박스)
  AFTER 결과: 실제 AI 생성 슬라이드 캡처 이미지
              「30枚・ウォーターマークなし・無料」 배지 3개
[수치] 「このユーザーの作成時間: 2分48秒」
[CTA] 「私もやってみる（無料）」
[하단 아이콘 3개] 워터마크 없음 / PPTX 무료 / 편집 자유
```

**HTML 포인트**: Before=회색 박스, After=실제 슬라이드 캡처로 비주얼 대비 극대화. "2분 48초"처럼 구체적 수치가 신뢰감 포인트.

## 아이디어 5 — Smart Block 마법 체험 메일
**타겟**: COMPANY / UNKNOWN  
**타이밍**: D+8~11 내삽  
**소구점 기반**: 경쟁사 없는 Smart Block (백서 강점 ⑥)

```
구조:
[헤더] 「内容を変えたら、レイアウトが崩れた経験ありませんか？」
[Before/After 2단 카드]
  Without Smart Block: 텍스트 늘렸더니 이미지 밀리고 정렬 무너진 슬라이드
  With Smart Block: 텍스트 늘려도 자동 재배치, 깔끔하게 유지
[설명 박스]
  「MiriCanvasのSmart Blockは、内容を変えるたびに
   レイアウトを自動で再計算します。崩れません。」
[솔직한 주의사항] ※ Smart Block解除はいつでも可能。一部操作はCtrl+Zが効かない場合があります。
[CTA] 「Smart Blockを試してみる」
```

**HTML 포인트**: Before=와장창 슬라이드, After=정갈한 슬라이드 나란히. 주의사항 솔직 표기 → 신뢰도 상승.

## 아이디어 6 — "첫 작품 완성" 활성화 메일 (가장 단순)
**타겟**: 전 세그먼트 공통 / 특히 D+3 기준 미활성 유저  
**타이밍**: D+3 (가입 후 미활성 조건 분기)  
**소구점 기반**: 첫 생성 경험 유저의 리텐션 압도적으로 높음

```
구조:
[헤더] 「最初の1枚を作ると、何かが変わります」
[진행도 바 CSS only]
  [가입] → [첫 생성 ★] → [편집] → [완성]
            ↑ あなたは今ここ
[카피]
  「データによると、最初の1枚を作ったユーザーの
   多くが翌日も戻ってきています。理由は簡単です。
   自分で作ったものは、見返したくなるから。」
[CTA 크게] 「1枚だけ作ってみる（3分）」
[서브 링크] 「テンプレートから選ぶ → 」
```

**HTML 포인트**: 최대한 단순 (텍스트 + CTA only). 진행도 막대 CSS only. CTA에 시간 명시("3分")로 허들 낮춤.
