# 작업 원칙

## UI 변경 시 파일 처리

UI를 수정할 때는 변경 규모에 따라 파일 처리 방식이 다름.

### 마이너 체인지 → 기존 파일을 그대로 수정
- 일부 컴포넌트 텍스트/색상 한두 군데 조정
- 작은 레이아웃 미세 조정
- 카피 / 라벨 수정
- 버그 픽스

### 전반적인 UI 변화 → **새 파일로 작성하고 기존 파일은 유지**
- 디자인 시스템 / 컬러 팔레트 전면 교체
- 타이포그래피 골격 변경 (예: 명조 → 산세리프)
- 레이아웃 구조 변경
- 컴포넌트 스타일 전반 재작업

**이유**: 비교/리뷰가 가능해야 하고, 롤백 여지를 남겨야 하며, 어떤 방향이 더 나은지 시각적으로 확인할 수 있어야 함.

**파일명 규칙**: `{원본명}_v2.html`, `_v3.html` 식으로 버전 suffix. 기존 파일은 절대 덮어쓰지 않음.

판단이 애매하면 사용자에게 확인.
