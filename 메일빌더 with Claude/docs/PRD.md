# CRM 메일 빌더 — 제품 요구사항 정의서 (PRD)

> **상태:** 설계 완료 (구현 대기)
> **작성:** 2026-04-25
> **목적:** 사용자가 레이아웃·박스·요소를 직접 조립해 이메일 HTML을 만드는 로컬 빌더
> **AI 역할:** 없음 — 순수 수동 빌더 (기존 `_pipeline/` AI 도구와 별개의 새 제품)
> **기술 스택:** Flask + 바닐라 JS + CDN 라이브러리 (Sortable.js, contentEditable 자체 툴바)

---

## #1. 프로젝트 개요

- 단일 사용자, 로컬 PC 전용 도구
- 백엔드 + 프론트엔드 = 같은 머신에서 동작
- 기존 `_pipeline/` 폴더는 건드리지 않음. 새 폴더 `_builder/`로 분리해서 신규 개발
- 출력물: 이메일 HTML 코드 (사용자가 직접 복사해 Braze 등 CRM 툴에 붙여넣음)
- 발송·외부 연동·DB 없음
- 지원 언어: 일본어 / 한국어 / 영어 / 포르투갈어(브라질)

## #2. 화면 구성 (1 + 3)

좌측 **고정 사이드바** + 우측 본문(상단 **3개 탭**)

| 영역 | 설명 |
|---|---|
| 사이드바 | 항상 노출. 제작한 메일 목록 |
| 소개탭 | 제품 컨셉·사용법. localhost 접속 시 기본 진입 화면 |
| 제작탭 | 메일 편집 작업 영역 |
| 비교탭 | 사이드바에서 "비교" 토글한 메일들을 비교 |

## #3. 사이드바

### 3.1 메일 row 구성
- **번호** (생성 순서로 영구 고정. 기존 메일 삭제돼도 다른 메일 번호 안 바뀜)
- **언어 라벨** (🇯🇵 / 🇰🇷 / 🇺🇸 / 🇧🇷)
- **짧은 타이틀** — 더블클릭으로 인라인 수정 모드, Enter로 확정
- 버튼 4개: **편집 / 비교 / 복제 / 삭제**

### 3.2 편집 버튼 (상태 토글)
- 클릭 시 ① 버튼 색·텍스트 활성화 ("편집 중") ② 제작탭으로 자동 이동 ③ 해당 메일 데이터 로딩
- 동시 편집 1개 제한 — 다른 메일의 편집 버튼 클릭 시 팝업: "현재 편집 중인 메일을 닫은 다음, 편집할 메일을 선택해주세요"

### 3.3 비교 버튼 (상태 토글, 다중 선택)
- 클릭 시 색·텍스트 변화 ("비교 중")
- 한 번 더 누르면 해제
- 비교 탭에서도 해제된 메일은 제외됨

### 3.4 복제 버튼 (v1 신규)
- 클릭 시 같은 내용으로 새 메일 생성. 제목 끝에 "(복사본)" 자동 추가
- 사이드바에 새 row 추가 (번호 자동 부여)

### 3.5 삭제 버튼
- 클릭 시 컨펌 팝업: "정말 이 메일을 삭제하시겠습니까?" → [네 / 아니요]
- "네" 시 메일 데이터·HTML 모두 제거 (백업도 함께 정리)

### 3.6 정렬·검색
- 정렬: 생성순 역순(최신 위) 고정
- 검색 기능: **v2** 후보

- localhost 첫 접속 시 기본 화면 (사이드바 + 소개탭)

## #5. 제작탭

### 5.1 기본 상태 (편집 중인 메일 없음)
- 중앙에 **"신규메일제작"** CTA 버튼 + 언어 선택 드롭다운
- 안내 문구: "사이드바에서 편집할 메일의 '편집' 버튼을 눌러주세요"
- 신규메일제작 클릭 시:
  - **14개 프리셋이 모두 한 번씩 등장하는 데모 메일** 생성 (Hero → Hero+subtitle → Section divider → ... → Footer Company → Footer Unsubscribe 순). 사용자가 불필요한 부분을 삭제·재배치하며 다듬는 흐름
  - 사이드바에 새 row 추가 (번호 자동 부여, 선택한 언어로 라벨)
  - 상단바·좌측 편집·우측 렌더링에 즉시 반영

### 5.2 편집 모드 — 상단바
| 요소 | 설명 |
|---|---|
| 번호·제목·제작일자·언어 | 좌측 |
| 제목 더블클릭 | 인라인 수정 |
| 저장 버튼 | 우측. 수정 내용 저장 |
| 닫기 버튼 | 우측. **저장이 완료되어야만 활성화** |
| 닫기 동작 | 제작탭 기본 상태로 복귀 + 사이드바 편집 버튼 색·텍스트 원복 ("편집") |

### 5.3 편집 모드 — 좌측 편집 화면 (2 분할)

**왼쪽 = 라이브러리 패널 (토글 가능, 기본 펼침)**
- 상단: [기본] / [프리셋] 탭
- 검색창 (한/영 키워드)
- 카테고리별 카드 (각 카테고리 ▾▸ 토글 — 사용자별 LocalStorage 저장)
- ⏴ 버튼으로 패널 전체 접기/펼치기

**오른쪽 = 구성 영역**
- 현재 메일의 레이아웃·박스·요소 카드들이 위→아래 스택
- 라이브러리에서 드래그 → 원하는 위치에 떨굼
- 각 카드 공통: 핸들로 순서 변경, ⊗ 삭제, ⚙ 여백 조정
- 편집 방식 (요소 종류별):
  - **텍스트**: 더블클릭 → contentEditable 인라인 편집 (리치 텍스트 툴바 부유)
  - **이미지**: 클릭 → 카드 내 펼쳐지는 속성 입력 (src, alt, width, 정렬, 옵션 링크)
  - **버튼**: 클릭 → 카드 내 속성 입력 (라벨, URL, 색상, 크기, 정렬)
  - **구분선**: 클릭 → 색·두께·너비
  - **여백**: 클릭 → 높이만
  - **레이아웃·박스**: 클릭 → 배경색·border (박스만)·패딩 패널

**계층 정책:**
- 비편집 (UI 미노출): doctype, html, head, body의 outer wrapper들 (메일 골격)
- **편집 대상**: 모든 레이아웃 / 박스 / 요소 (3계층). 푸터 2종도 일반 프리셋으로 자유 편집·삭제·재배치 가능 (사용자 책임. 실수 시 Undo 또는 다른 메일에서 복사)
- **중첩 정책**: 레이아웃은 최상위 고정 (nesting X). 박스는 자유 중첩 (Box-in-Box OK). 요소는 leaf

### 5.4 편집 모드 — 우측 렌더링 화면
- 좌측 변경사항이 **실시간 반영** (200ms debounce)
- 상단 토글: **비주얼 / 코드**
  - 비주얼: 실제 렌더링된 이메일 모습
  - 코드: HTML 소스 텍스트 (textarea readonly + 복사 버튼)

### 5.5 리치 텍스트 편집 (요소: 텍스트)
- contentEditable + 자체 툴바
- 화이트리스트: `<strong>`, `<u>`, `<br>`, `<a href>`, `<ul>/<ol>/<li>`, 헤딩 토글(H1-H4)
- 그 외 모든 태그·인라인 스타일은 저장 시 sanitize로 제거 → 이메일 클라이언트 호환성 100%

## #6. 비교탭

- 사이드바에서 "비교" 토글한 메일들을 모음
- **레이아웃: 가로 스크롤** — 메일 카드들이 옆으로 나열, 화면 폭 부족 시 좌우 스크롤
- 메일 카드 폭은 **이메일 표준 600px 고정**이 기본
- **상단 툴바에 폭 조절 UI:**
  - 🖥️ 데스크톱 (600px)
  - 📱 모바일 (375px)
  - ↔️ 맞춤 (슬라이더)
  - 변경 시 모든 메일 카드 폭이 동시에 변함 (모바일 렌더링 동시 비교용)
- 비교 탭 내에서 메일을 "비교 중" 해제 시 즉시 빠짐

## #7. 라이브러리 구조

### 7.1 [기본] 탭 — 빌딩 블록 (총 11종)

#### 레이아웃 5종 (구조·열 분할)
| # | 종류 | 설명 |
|---|---|---|
| 1 | 1열 | 단일 콘텐츠 (가장 흔함) |
| 2 | 2열 50:50 | 좌우 균등 분할 |
| 3 | 2열 33:67 | 작은 좌 + 큰 우 |
| 4 | 2열 67:33 | 큰 좌 + 작은 우 |
| 5 | 3열 33:33:33 | 3등분 |

레이아웃 공통 속성: 4방향 패딩 + 배경색 + 모바일 자동 1열 스택

#### 박스 1종
- 색·border·radius·padding 자유 조절 가능한 wrapper
- 박스 안에 박스 자유 중첩 가능

#### 요소 5종 (콘텐츠 원자)
| # | 종류 | 속성 |
|---|---|---|
| 1 | 텍스트 (Text) | 리치 텍스트 (H1-H4 토글, 단락, 리스트), 정렬·색상·사이즈 |
| 2 | 이미지 (Image) | src, alt, width, 정렬, 옵션 링크 |
| 3 | 버튼 (Button) | 라벨, URL, 배경색, 크기(소/중/대), 정렬 |
| 4 | 구분선 (Divider) | 색상, 두께, 너비 |
| 5 | 여백 (Spacer) | 높이만 |

(소셜 모듈은 v2 후보로 이관)

### 7.2 [프리셋] 탭 — 자주 쓰는 조합 (14종)

| # | 이름 | 구성 |
|---|---|---|
| 1 | Hero | 1열 + 이미지 + 텍스트(H1) |
| 2 | Hero with subtitle | 1열 + 이미지 + 텍스트(H1) + 텍스트(P) |
| 3 | Section divider | 1열(bg light) + 텍스트(H2) |
| 4 | Highlighted intro | 1열(bg light) + 박스(white) + 텍스트 |
| 5 | Body block | 1열 + 텍스트(H2) + 텍스트(P) + 이미지 |
| 6 | CTA solo | 1열 + 버튼 |
| 7 | CTA + microcopy | 1열 + 텍스트(소형) + 버튼 |
| 8 | CTA + body | 1열 + 텍스트(P) + 버튼 |
| 9 | CTA emphasized | 1열 + 텍스트(H2) + 텍스트(P) + 버튼(대) |
| 10 | Card full | 1열 + 텍스트(H3) + 이미지 + 버튼 |
| 11 | Card 2-up | 2열 50:50 + 각 칸에 이미지+텍스트+버튼 |
| 12 | Side topic box | 1열(bg light) + 박스(white) + 텍스트 + 이미지 + 버튼 |
| 13 | Footer Company | 1열(bg dark #2a2a2a) + 텍스트(흰 글씨, 회사명·HP·이메일·주소) |
| 14 | Footer Unsubscribe | 1열(bg dark or footer 연속) + 텍스트(수신거부 링크 + 안내 문구. Braze 변수 `{{${set_user_to_unsubscribed_url}}}` 그대로 보존) |

**프리셋 동작:** 드롭하면 레이아웃·박스·요소 조합으로 분해되어 들어감. 추가 후엔 일반 빌딩 블록처럼 자유 편집·삭제·재배치 가능. 프리셋 자체는 별도 데이터 타입이 아님 — 그냥 "한 번에 여러 개 추가" 단축키.

**Braze 변수 보존:** Footer Unsubscribe 프리셋의 텍스트 요소에 들어있는 `{{${set_user_to_unsubscribed_url}}}` 토큰은 sanitizer 화이트리스트에 추가하여 그대로 출력되도록 처리.

## #8. 여백 컨트롤

- 각 레이아웃·박스·요소 카드에 **⚙ 여백** 패널 (접기/펼치기)
- **레이아웃·박스**: 4방향 직접 입력 (위/아래/왼쪽/오른쪽)
- **요소**: 위/아래 마진만 (좌우는 부모가 결정)
- **↻ 기본값** 버튼 — 라이브러리 디폴트로 즉시 복귀
- 저장 형식: 디폴트와 같으면 JSON에서 생략 → 라이브러리 원본과 100% 동일 렌더

## #9. 백업 & Undo

### 백업 (UI 없음)
- 저장 시 직전 파일이 `mails/.backup/{id}-{timestamp}.json`로 이동
- 최근 3개만 보관, 그 외는 자동 삭제
- 복구는 사용자가 Claude Code에 명령 (예: "메일 4번 어제 저녁 백업으로 복구")

### Undo
- Ctrl+Z 30단계
- 메일 닫으면 스택 초기화
- 저장 여부와 무관하게 동작 (저장 후에도 Undo 가능)
