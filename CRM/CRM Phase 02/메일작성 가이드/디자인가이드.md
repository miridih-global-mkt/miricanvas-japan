## §3. 인지심리 원칙 (카피 & 구조에 적용)

### 전주의적 처리 (Pre-attentive Processing)
- 색상으로 계층 표현: 민트(행동), 네이비(중요), 회색(부가)
- 크기로 중요도: H1 → Body 명확한 단계 구분
- 여백 = 내용만큼 중요. 밀도 높은 이메일 → 이탈

### 폰 레스토프 효과 (Von Restorff)
- 핵심 메시지 1개만 색상/크기로 구분
- 2개 이상 강조하면 효과 없음

### 인지 부하 최소화
- 이메일당 CTA 1개 (2개 넘으면 어느 것도 클릭 안 함)
- 단락당 3~4문장
- 섹션당 1개 포인트

### 계열 위치 효과 (Serial Position)
- 첫 2~3행에서 클릭 여부 결정
- 헤더 카피가 전체 이메일의 성패를 좌우

## §4. 이메일 유형별 구조 원칙

### 온보딩 이메일 (현재 주력)
```
구조: 문제/기회 → 해결책 → CTA → 부가 정보
길이: 300~500단어 (일본어 기준)
CTA: 1개, 상위 fold 안에
이미지: 1~2개 (없어도 OK)
```

### 기능 소개 이메일
```
구조: 기능명 → Before/After → 사용법 → CTA
주의: 기능 "나열" 금지. 1개 기능에 집중.
비주얼: 기능 사용 GIF 또는 스크린샷
```

### 재활성화 이메일
```
구조: 공감 → 해결책(쉬운 첫 단계) → CTA
톤: 부드럽게, 압박 없이
CTA: "〜から再開する" (재개 느낌)
```

## §5. 일본 시장 특화 규칙

### 문화적 특성
- **오모테나시 (おもてなし)**: 관계 형성 우선. 즉각 전환 압박 금지.
- **정보 밀도**: 글로벌 이메일보다 텍스트 비중 높아도 됨 (70:30 텍스트:이미지 허용)
- **계절 감수성**: 시즌 인사 → 관계 유지 뉴스레터에서 활용
- **신뢰 우선**: 수치 + 사례 + 회사 정보 충분히 제공

### 타이밍
```
최적 발송 시간: 화~목 오전 9~11시 JST
피할 시간: 금요일 오후, 월요일 오전, 연휴 직전
오픈율 기준: 일본 평균 31.7% (글로벌 23%). 성과 기준 높게 설정.
```

### 톤 원칙
```
✅ 간접 권유: 「ぜひお試しください」「いかがでしょうか」
✅ 정중체: です・ます체 기본
✅ 계절 인사 도입 (뉴스레터 한정)
❌ 직접 명령: 「今すぐ〜してください」
❌ 과도한 긴급성: 「今だけ！」「残りわずか！」(과장)
❌ 캐주얼 체: だ・である体 (B2C에서도 온보딩엔 사용 금지)
```
