# MiriCanvas Japan Email Design System v2
작성일: 2026-05-21

이 문서는 어떤 이메일을 만들 때도 적용하는 공통 프로토콜이다.
"무엇을 만드나"가 아니라 "어떻게 만드나"를 정의한다.

---

## 0. 설계 결정의 근거

각 수치와 규칙은 감각이 아니라 다음 리서치에서 도출했다.

| 결정 | 근거 |
|---|---|
| H1 34px | Grammarly·Typeform·Figma 이메일 H1 실측 36~40px. 일본어는 자형이 넓어 동일 pt에서 더 커 보이므로 34px로 보정. |
| 섹션 수직 여백 40px | Typeform·Loom 이메일 섹션 간격 실측 40~48px. "여백은 독자에 대한 배려"(読み手への配慮) — 일본 이메일 트렌드 리서치. |
| `word-break: break-all` 인라인 강제 | 일본어 장문은 body CSS 상속만으로 브라우저/클라이언트 간 불일치 발생. 인라인 명시 필수. |
| `line-height: 1.7` 인라인 강제 | 일본어 가독성 기준값. freee·SmartHR 이메일 측정값 1.6~1.8. body 상속만으로는 일부 클라이언트에서 초기화됨. |
| Primary CTA 패딩 16px 56px | 최소 터치 타깃 44px × 가로 여유. Grammarly CTA 실측 기준. |
| 이미지 radius-md(8px) | Figma·Canva 이메일 이미지 처리 기준. 부드럽지만 과하지 않은 라운딩. |
| 버튼·박스 radius-sm(4px) | 카드보다 작은 UI 요소는 더 작은 라운딩으로 시각적 계층 유지. |

## 1. 설계 원칙

1. **시스템이 판단한다** — 색·크기·여백·모듈 선택은 이 문서의 규칙에서 나온다. 규칙 없는 새 케이스는 추가하고 기록한다.
2. **CTA 하나로 수렴한다** — Primary CTA 1개. 모든 요소는 그 CTA를 향한 시선 경로다. 강조가 많으면 아무것도 강조되지 않는다.
3. **여백은 설계다** — 특히 일본어 본문은 자형 밀도가 높아 여백이 가독성에 직결된다.

## 2. 폰트 토큰

### 폰트 패밀리
```
font-ja: 'Hiragino Kaku Gothic ProN', 'Hiragino Sans', 'Meiryo', sans-serif
```

| 폰트 | 환경 | 이유 |
|---|---|---|
| Hiragino Kaku Gothic ProN | macOS / iOS | Apple 기기 표준 일본어 고딕. 가장 우선. |
| Hiragino Sans | macOS 최신 | ProN의 대체·확장판. |
| Meiryo | Windows | Windows 표준 일본어 웹 폰트. |
| sans-serif | 기타 | 최종 폴백. |

### 폰트 굵기
| 토큰 | 값 | 용도 |
|---|---|---|
| `weight-regular` | 400 | 본문, 보조 텍스트 |
| `weight-bold` | 700 | H1·H2·CTA·`<strong>`. 600(semi-bold) 사용 안 함 — 일본어 웹폰트에서 렌더링 불안정. |

## 3. 타이포그래피 스케일

| 레벨 | 크기 | 굵기 | 행간 | 자간 | 용도 |
|---|---|---|---|---|---|
| `text-h1` | **34px** | 700 | 1.3 | 기본 | 이메일 제목. 1개/메일 고정. 모바일 24px. |
| `text-h2` | 22px | 700 | 1.4 | 기본 | 섹션 제목, Q&A 질문. 최대 4개/메일. |
| `text-h3` | 18px | 700 | 1.4 | 기본 | 카드 제목, 박스 타이틀. |
| `text-body` | 16px | 400 | **1.7** | 기본 | 주 본문. `word-break: break-all` 인라인 필수. |
| `text-sub` | 14px | 400 | 1.6 | 기본 | 카드 설명, 보조 본문. |
| `text-caption` | 13px | 400 | 1.6 | 기본 | 푸터 회사정보. |
| `text-legal` | 12px | 400 | 1.6 | 기본 | 수신거부 문구. |
| `text-microcopy` | 15px | 700 | 1.4 | 기본 | CTA 직전 유도 1줄. |
| `text-cta` | 16px | 700 | 1 | 기본 | CTA 버튼 텍스트. |
| `text-step` | 16px | 700 | 1 | 기본 | STEP 번호 + 타이틀. |

### 규칙
- 모든 `<p>` 태그에 `line-height: 1.7; word-break: break-all` **인라인 명시** (body 상속 불신뢰)
- 모바일 H1: `.h1-title { font-size: 24px !important; }` (미디어쿼리)
- H1 정렬: T-1~5·7·8 → `center` / T-6·9·10 → `left`
- CTA 텍스트: **동사+목적어** 필수. "こちら" 단독 금지.

## 5. 여백 시스템

기준 단위: **8px**

| 토큰 | 값 | 용도 |
|---|---|---|
| `space-1` | 8px | 인라인 최소 간격 |
| `space-2` | 16px | 요소 내부 간격 (제목↔본문, 이미지↔캡션) |
| `space-3` | 24px | **좌우 콘텐츠 여백 고정** |
| `space-4` | 32px | 소섹션 상하 패딩 |
| `space-5` | 40px | **모듈 간 기본 상하 여백** |
| `space-6` | 48px | 히어로 상하 패딩 |

### 적용 기준
```
좌우 여백 (전 모듈 고정): padding-left: 24px; padding-right: 24px;
히어로:                   padding: 48px 24px 40px 24px;
모듈 기본:                padding: 40px 24px;
모듈 상단 연결:           padding: 0 24px 40px 24px;
하이라이트 박스 내부:     padding: 20px;
CTA 버튼:                 padding: 16px 56px;
푸터:                     padding: 40px 24px;
```

## 6. 라디우스 토큰

| 토큰 | 값 | 적용 대상 |
|---|---|---|
| `radius-sm` | **4px** | CTA 버튼, 하이라이트 박스, 스텝 헤더, 비교 테이블 외곽 |
| `radius-md` | **8px** | 카드 컨테이너, 이미지, 비디오 썸네일, box 모듈 흰 카드 |

> **규칙**: 같은 문맥의 요소는 같은 radius를 쓴다.
> 이미지는 항상 radius-md(8px). 버튼·박스·인라인 요소는 항상 radius-sm(4px).

## 7. CTA 위계

### 3단계
| 레벨 | 스타일 | 용도 | 개수 |
|---|---|---|---|
| **Primary** | `#26c7d9` 채운 버튼, 흰 텍스트 | 메일의 핵심 액션 | **1개/메일** |
| **Secondary** | 아웃라인 `border: 2px solid #26c7d9`, `#26c7d9` 텍스트 | 카드·스텝 내 보조 액션 | 최대 2개 |
| **Minimal** | 텍스트 링크 `color: #26c7d9; text-decoration: underline` | 저강도 유도 | 제한 없음 |

### Primary CTA HTML 표준
```html
<table border="0" cellpadding="0" cellspacing="0" align="center" style="margin: 0 auto;">
  <tr>
    <td align="center" style="background-color: #26c7d9; border-radius: 4px;">
      <a href="[URL]" target="_blank"
         style="display: inline-block; padding: 16px 56px;
                font-family: 'Hiragino Kaku Gothic ProN','Hiragino Sans','Meiryo',sans-serif;
                font-size: 16px; font-weight: bold; color: #ffffff;
                text-decoration: none; text-align: center; line-height: 1;
                -webkit-text-size-adjust: none;">
        動詞＋目的語
      </a>
    </td>
  </tr>
</table>
```

### Secondary CTA HTML 표준
```html
<table border="0" cellpadding="0" cellspacing="0">
  <tr>
    <td style="border: 2px solid #26c7d9; border-radius: 4px;">
      <a href="[URL]"
         style="display: inline-block; padding: 12px 28px;
                font-family: 'Hiragino Kaku Gothic ProN','Hiragino Sans','Meiryo',sans-serif;
                color: #26c7d9; text-decoration: none;
                font-size: 14px; font-weight: bold; line-height: 1;">
        テキスト
      </a>
    </td>
  </tr>
</table>
```

## 9. 금지 사항

| 금지 | 이유 |
|---|---|
| Primary CTA 2개/메일 (T-5 제외) | 핵심 액션이 모호해진다 |
| 하이라이트박스 3개 이상 | 모든 강조 = 강조 없음 |
| T-10에서 하이라이트박스 | 관계메일에 판촉 요소 = 신뢰 저하 |
| CTA에 "こちら" 단독 | 어디로 가는지 알 수 없다 |
| `<p>`에 line-height·word-break 생략 | 클라이언트별 초기화로 깨짐 |
| `mso-table-lspace` 중복 선언 | rspace와 분리 필수 |
| `lang="ko"` | lang="ja" 고정 |
| Liquid 사용 | 예외: `{{content_blocks.${...}}}` · `{{${set_user_to_unsubscribed_url}}}` 만 허용 |
