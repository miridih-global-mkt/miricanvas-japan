# 메트릭 선정 의사결정 근거 v7

작성: 2026-05-19. v4 시대 선정(9개)을 v7 ranking + redundancy·차원 분석에 따라 갱신.

## Executive Summary

CRM 메일 효과 측정용 핵심 메트릭을 **3 tier로 재정의**:

| Tier | 명칭 | 개수 | 용도 |
|---|---|---:|---|
| 🥇 | **최중요 ⭐⭐⭐** | 2 | 단일 KPI 보고, 임원/외부 1줄 요약 |

v4 시대의 "9개 핵심·2 ⭐⭐⭐" 대체. 일관된 v7 파이프라인(Welch + Winsorize 9) 기반 ranking 산출.

## 1. 메트릭 선정 가이드라인 (재확인)

- **좋은 메트릭**: 메일 효과가 있을 때 명확히 반응하고, 없을 때 흔들리지 않는 메트릭. 신호 대 잡음 비율(z값)이 큼
- **1순위 일관성**: 일별 신호 부호 일관성 ≥ 70%
- **2순위 명료성**: \|z\| ≥ 2 이상이면 잡음 대비 신호 충분
- **양·음 무관**: 양효과 측정뿐 아니라 역효과·중립 모두 잘 가르는 게 좋은 metric

### 평가 점수 (composite_score)

```
composite_score = 2 × (강신호 cell 수) + 1 × (중신호 cell 수)
강신호: |z| ≥ 3 + cons ≥ 0.7
중신호: 2 ≤ |z| < 3 + cons ≥ 0.7
```

cell = (canvas × user_type × treatment_step) 단위. 6 캔버스 × 7 user_type × ~6 step = ~150 cell.

### 데이터·계산 방법

- 파이프라인: v7
- z 산출: Welch two-sample (user-level SE)
- 9 metric에 Winsorize 99% 적용 (DPU·Stickiness·LPU_total)
- 자세히: `파이프라인_원칙.md`

## 2. Redundancy 분석 — 왜 시리즈에서 1~2개만 골랐나

44개 metric 중 약 30개는 "동일 차원을 시간 윈도우만 다르게 측정"하는 시리즈:

| 시리즈 | 멤버 | redundancy 평가 |
|---|---|---|
| **LPU** (로그인 평균 일수) | 24h, 72h, 168h, total | 매우 높음 (시간 차이만) |
| **LRate** (로그인 도달 binary) | 24h, 72h, 168h, total | LPU와 거의 같음 (binary 버전) |
| **L_ge2/ge3** (로그인 깊이) | 72h_ge2, 168h_ge2, total_ge2, 168h_ge3, total_ge3 | 중간 (임계치+시간 조합) |
| **DPU** (디자인 평균 개수) | 24h, 72h, 168h, total | 매우 높음 |
| **DRate, D_ge2/ge3** (디자인 binary·깊이) | 다수 | DPU와 중복 |
| **LD** (로그인&디자인 binary) | 24h, 72h, 168h, total | LDeep이 더 깊은 신호 |
| **Stickiness** (로그인일당 디자인) | 24h, 72h, 168h, total | 매우 높음 |

→ **각 시리즈에서 1~2개만 선택**. 시간 윈도우 다양성(D1·D3·D7) 보존 차원에서 필요한 경우만 2개.

## 4. 선정된 Tier

### 🥇 최중요 ⭐⭐⭐ — 2개

| metric | 한국어 | composite | 차원 | 단일 KPI 적합도 |
|---|---|---:|---|---|
| **Stickiness_72h** | 로그인일당디자인수D3 | **78** (1위) | 활성 강도 | 메일의 창작 집중도 변화 측정에 강함 |
| **LPU24h** | 로그인평균일수D1 | **73** (3위) | 로그인 빈도 | 가장 많은 cell에 일관 신호 (n_cells 83) |

- 빈도(LPU) — "메일 후 더 자주 들어옴?"
- 강도(Stickiness) — "들어와서 더 활발히 작업함?"

| metric | 한국어 | composite | 차원 |

## 5. v4 → v7 변동 요약

| 변동 | 내용 |
|---|---|
| **新 ⭐⭐⭐** | Stickiness_72h, LPU24h (LPU168h, L168h_ge3 → 강등) |
| **신규 채택** | Stickiness_72h, LPU24h, LDeep168h, LRate168h, DPU72h, L72h_ge2 |
| **제외** | LPU_total, DPU168h, LRate24h |
| **유지** | LPU72h, L72h_ge2, DPU(D1 ver), LDeep, LRateReturn(anchor만) |
| **시리즈 redundancy 정리** | LRate 시리즈 1개만(D7), DRate 시리즈 모두 제외, LD 시리즈 모두 제외(LDeep로 대체) |

### v4 시대 ⭐⭐⭐가 강등된 이유

- **LPU168h**: v4의 z=10~30은 weekly cohort SE 부풀림 효과. v7 Welch에서 실제 magnitude는 z=1~2 수준. 14위지만 시간 윈도우 D7 cover 차원에서 🥉에 유지
- **L168h_ge3**: 동일 이유. 강신호 0건, 중신호 11건. 깊이 D7 차원 보존 차원에서 🥉에 유지

이는 metric 자체 변화가 아니라 **v4 → v7의 z 계산 정확도 향상**으로 인한 재평가.

## 6. 운영 방침

### 의사결정 임계

- **강한 양효과**: \|z\| ≥ 3 + cons ≥ 0.7 → 즉시 채택 검토
- **강한 역효과**: \|z\| ≥ 3 + cons ≥ 0.7 → 즉시 배제 또는 정성 분석
- **약신호**: 1 ≤ \|z\| < 2 → 모니터링, 데이터 누적 대기

### 보고 형태

- **단일 KPI**: 🥇 ⭐⭐⭐ 2개 중 보고 목적 따라 선택

- **상세 리포트 (HTML/PDF)**: 🥉 11개
- **애드혹 분석**: 백엔드 44개 measures 활용
