폴더 구조

crm/
├── _pipeline/                     # 기존 AI 도구 — 건드리지 않음
├── module_catalog_annotated.html  # 참조 자료
├── docs/PRD.md                    # 방금 저장
└── _builder/                      # 이번 작업 ← 여기 새로 만듦
    ├── app.py                     # Flask 진입점
    ├── requirements.txt
    ├── server/
    │   ├── routes.py              # API 라우트
    │   ├── storage.py             # JSON 파일 저장/로딩
    │   ├── library.py             # 섹션·모듈 정의 (하드코딩)
    │   └── renderer.py            # JSON → 이메일 HTML
    ├── static/
    │   ├── index.html             # 단일 SPA 엔트리
    │   ├── css/style.css
    │   └── js/
    │       ├── app.js             # 메인 컨트롤러 (탭 전환·상태)
    │       ├── sidebar.js
    │       ├── editor.js          # 좌측 편집 화면
    │       ├── preview.js         # 우측 비주얼/코드 토글
    │       └── compare.js
    └── mails/                     # 사용자 데이터 (자동 생성)
        └── {id}.json

핵심 결정:
① 백엔드 = Flask가 정적 파일(HTML/CSS/JS) 서빙 + JSON API 제공
② 프론트엔드 = 단일 SPA (페이지 1개, JS로 탭 전환). 사이드바 상태가 탭 사이에서 유지됨
③ 메일 데이터 = 1메일 = 1 JSON 파일 (DB 없음, 사람이 읽을 수 있는 형식)
④ 포트 = 5001 (기존 _pipeline/의 5000과 충돌 방지)
