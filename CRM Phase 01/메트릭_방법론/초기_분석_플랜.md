# 캔버스 별 성과: "가장 좋은 메트릭" 선정

## Context

`temp.ejjeong_temp.crm_global_new` 테이블 기반 CRM Global Canvas Dashboard의 **(개괄) 캔버스 별 성과** 탭은 캔버스×스텝별로 20여 개의 메트릭을 한 번에 보여준다.
사용자(CRM 담당)의 목표:
- **무엇을 결정하는가**: Control 대비 Treatment의 uplift/downlift가 **우연이 아닌 지속적 경향**으로 잡히는지
- **궁극 목적**: 1·2·4주 뒤에도 **계속 쓰는 유저(=로그인 또는 디자인 생성)** 만들기
- **대상**: 신규 유저 온보딩/활성화

---

## 1. "좋은 메트릭"의 5가지 기준

| # | 기준 | 왜 중요한가 |
|---|---|---|
| **S1** | **북극성 정렬 (Validity)** | 1·2·4주 retention과 인과·상관 높아야 함 |
| **S2** | **통계적 안정성 (Signal-to-Noise)** | Control vs Treatment 차이가 표본 분산보다 커야 함 |
| **S3** | **민감도 (Sensitivity)** | 캠페인 효과가 작아도 움직임을 포착. floor/ceiling 효과 없어야 함 |
| **S4** | **시간 적합성 (Timeliness)** | 결정 주기 안에 충분한 표본이 쌓여야 함 |
| **S5** | **해석·게이밍 저항 (Interpretability)** | 팀·이해관계자가 단위·방향을 직관적으로 이해. 이상치에 흔들리지 않음 |

---

## 2. 기존 22개 메트릭 평가 요약

**핵심 발견**:
- **DRate168h**: S1~S5 모두 최상위. **Primary 후보 1순위**
- **상한DPU**: depth(얼마나 깊이 쓰는지)를 outlier-robust하게 측정
- **D>3**: "7일 내 ≥3"으로 재정의 시 최고의 retention proxy
- **Stickiness**: 분자·분모 둘 다 변동 → 해석 모호. **제외 권장**
- **×100 스케일 메트릭(LPU, DPU)**: 단위 함정. raw 평균으로 재정의 필요

---

## 3. 최종 추천 — 2-layer 메트릭 체계

### Layer 1: Primary (단일 KPI)
> ⭐ **DRate168h** (7일 내 디자인 ≥1 도달률)
- 이유: S1~S5 종합 최상. binary라 통계 안정. 신규 활성화 표준 정의.
- **"디자인 생성이 primary"** 원칙에 가장 직접 부합.

### Layer 2: Triangulation (1차에서 lift 잡힌 후 검증)
1. **D7≥3** (신규 정의) — *활성화 깊이*: 1회 사용 vs 반복 사용
2. **상한DPU** — *활성화 강도*: 사용량 평균 (outlier 보정)
3. **LRate168h** — *간접 proxy*: 디자인 안 만들어도 편집·다운로드한 유저까지 포함

### Guardrail
- **user (count distinct)**: SRM 체크
- **open_rate**: 메일 deliverability 이상 감지
- **Long-term: LRateTotal / DRateTotal**: 7일 lift가 장기 reversal 없는지 후행 검증

### 결정 주기 추천: 2-tier cadence
- **일별(매일)**: DRate168h의 **7일 rolling cohort lift** 추적
- **주별(매주)**: 누적 Control vs Treatment 비교 + statistical test
- **월별**: LRateTotal·DRateTotal로 7일 결과의 장기 유효성 검증

> **"24h를 매일 보는 것"은 비추**: 표본이 작아 noise 큼. 168h를 매일 rolling으로 보는 게 신호 안정성·신선도 균형.
